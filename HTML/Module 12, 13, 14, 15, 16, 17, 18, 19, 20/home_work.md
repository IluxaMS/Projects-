### Практическое задание: «Страница регистрации на IT-конференцию»

Необходимо сверстать одностраничный сайт (HTML + CSS), посвященный регистрации на конференцию.

#### 1. Структура и контент (HTML)

**Шапка сайта:**
*   Название мероприятия.
*   Навигационное меню, пункты которого являются якорными ссылками, ведущими к разделам страницы: «О конференции», «Спикеры», «Расписание», «Регистрация».
*   Контактные данные: ссылка на email и ссылка на телефонный номер.

**Раздел «О конференции»:**
*   Главный заголовок страницы.
*   Текстовое описание мероприятия (несколько абзацев).
*   В тексте должна присутствовать короткая цитата, встроенная в предложение.
*   В тексте должна присутствовать длинная оформленная цитата отдельным блоком.
*   Одно ключевое слово или фраза в тексте должны быть визуально выделены (акцент).
*   Раздел должен заканчиваться горизонтальной линией-разделителем.

**Раздел «Спикеры»:**
*   Заголовок раздела.
*   Две карточки спикеров. Каждая карточка содержит:
    *   Фотографию.
    *   Имя и Фамилию.
    *   Должность/Компанию.
    *   Список тем докладов.
    *   Контактные данные (например, email или телефон), где часть контактов находится на одной строке, а часть перенесена на новую.

**Раздел «Расписание»:**
*   Заголовок раздела.
*   Таблица с программой мероприятия.
*   Таблица должна иметь заголовок, шапку, тело и подвал.
*   В таблице должны быть объединенные ячейки:
    *   Как минимум одна ячейка, объединяющая несколько столбцов.
    *   Как минимум одна ячейка, объединяющая несколько строк.

**Раздел «Регистрация»:**
*   Заголовок раздела.
*   Форма регистрации, разбитая на две логические группы полей с заголовками групп.
*   **Группа 1 (Личные данные):**
    *   Поле для ввода имени.
    *   Поле для ввода фамилии.
    *   Поле для ввода email.
    *   Поле для ввода телефона.
    *   Поле для ввода пароля.
    *   Поле для ввода ссылки на портфолио.
*   **Группа 2 (Детали участия):**
    *   Выбор типа билета (вариант выбора только одного из нескольких).
    *   Выбор даты прибытия.
    *   Выбор времени звонка.
    *   Выбор месяца подписки.
    *   Выбор недели мероприятия.
    *   Выбор даты и времени сеанса.
    *   Поле для ввода числа гостей.
    *   Поле поиска с выпадающими подсказками (список вариантов).
    *   Поле для загрузки файла.
    *   Многострочное поле для комментариев.
*   Кнопки управления формой:
    *   Кнопка отправки данных.
    *   Кнопка очистки формы.
    *   Одна из кнопок должна получать фокус автоматически при загрузке страницы.
    *   Одна из кнопок должна быть неактивной.

**Боковая панель:**
*   Ссылка на профиль в Skype (открывается в новой вкладке).
*   Список дополнительных полезных ссылок.

**Подвал сайта:**
*   Физический адрес организаторов.
*   Информация об авторских правах.

#### 2. Стилизация (CSS)

**Общие требования:**
*   Использовать предоставленный файл обнуления стилей.
*   Все классы должны быть названы в соответствии с методологией БЭМ.

**Типографика и оформление текста:**
*   Основной шрифт страницы должен быть без засечек.
*   Размер основного текста должен быть задан в пикселях.
*   Цвет основного текста должен быть темно-серым.
*   Межстрочный интервал должен быть увеличен для удобства чтения.
*   Все заголовки первого уровня должны быть написаны заглавными буквами и иметь увеличенное расстояние между символами.
*   Заголовки второго уровня должны быть выровнены по центру.
*   Заголовки третьего уровня должны иметь полужирное начертание.
*   Главный заголовок должен иметь легкую тень.
*   Первый абзац текста в разделе «О конференции» должен иметь красную строку (отступ первой строки).
*   Текст в блоке длинной цитаты должен иметь увеличенное расстояние между словами.
*   Ссылки во всем документе должны иметь подчеркивание точками.
*   Цвет ссылок должен отличаться от основного цвета текста.

**Селекторы и специфичность:**
*   Должны быть использованы селекторы по тегу, по классу и по ID.
*   Должен быть использован селектор, обращающийся к нескольким элементам одновременно (через запятую).
*   Должен быть использован селектор потомка (пробел) и селектор дочернего элемента первого уровня (>).
*   Должна быть продемонстрирована работа специфичности: стиль, заданный через класс, должен переопределять стиль, заданный через тег, для одного и того же элемента.

**Единицы измерения:**
*   Ширина основных контейнеров должна быть задана в относительных единицах (проценты или viewport units).
*   Размеры шрифтов для второстепенных элементов (подписи, мелкие детали) должны быть заданы в `em` или `rem`.
*   Высота некоторых блоков (если таковые будут использоваться для структуры) может быть задана в `vh`.

**Запреты:**
*   Не использовать свойства, не входящие в список изученных (в частности, `margin`, `padding` для создания отступов между блоками, `border`, `display`, `flex`, `grid`).
*   Не использовать псевдоклассы (`:hover`, `:focus` и др.) и псевдоэлементы.
