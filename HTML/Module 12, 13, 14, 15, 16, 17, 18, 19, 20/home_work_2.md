### Практическое задание: «Страница подкаста и радиостанции»

Необходимо сверстать страницу интернет-радиостанции или подкаста, используя изученные технологии.

#### 1. Структура и контент (HTML)

**Шапка сайта:**
*   Название радиостанции.
*   Навигационное меню, пункты которого являются якорными ссылками, ведущими к разделам страницы: «Эфир», «Ведущие», «Расписание», «Подписка».

**Раздел «В эфире»:**
*   Главный заголовок страницы.
*   Блок с текущим выпуском, содержащий аудиоплеер и обложку выпуска с подписью.
*   Текстовое описание текущего выпуска, состоящее из нескольких абзацев.
    *   В тексте должна присутствовать короткая цитата, встроенная в предложение.
    *   В тексте должна присутствовать длинная оформленная цитата отдельным блоком.
    *   Одно ключевое слово или фраза в тексте должны быть визуально выделены.
*   Горизонтальная линия, отделяющая описание от следующего блока.

**Раздел «Ведущие»:**
*   Заголовок раздела.
*   Две карточки ведущих. Каждая карточка содержит:
    *   Фотографию ведущего.
    *   Имя и Фамилию.
    *   Описание стиля ведения эфира.
    *   Список любимых музыкальных жанров.

**Раздел «Расписание эфиров»:**
*   Заголовок раздела.
*   Таблица расписания, содержащая заголовок, шапку, тело и подвал.
*   В таблице должны присутствовать объединенные ячейки:
    *   Как минимум одна ячейка, объединяющая несколько столбцов.
    *   Как минимум одна ячейка, объединяющая несколько строк.
*   В подвале таблицы должна содержаться итоговая информация.

**Раздел «Подписка на рассылку»:**
*   Заголовок раздела.
*   Форма подписки, разбитая на две логические группы полей с заголовками групп.
*   **Группа 1 (Контактные данные):**
    *   Поле для ввода имени.
    *   Поле для ввода email.
    *   Поле для ввода телефона.
*   **Группа 2 (Настройки подписки):**
    *   Выбор частоты получения писем (возможность выбрать только один вариант из нескольких).
    *   Выбор даты начала подписки.
    *   Выбор времени получения письма.
    *   Поле поиска по архиву выпусков с выпадающими подсказками (минимум 3 варианта).
    *   Многострочное поле для пожеланий редакции.
*   Кнопка «Подписаться», которая получает фокус автоматически при загрузке страницы.
*   Кнопка «Очистить», которая является неактивной.

**Боковая панель:**
*   Видеоролик с анонсом следующего сезона, имеющий постер, элементы управления и воспроизводящийся автоматически без звука.
*   Подпись к видеоролику.
*   Список ссылок на социальные сети, открывающихся в новой вкладке.

**Подвал сайта:**
*   Адрес студии.
*   Информация об авторских правах.

#### 2. Стилизация (CSS)

**Общие требования:**
*   Использовать файл обнуления стилей.
*   Использовать методологию БЭМ для именования классов.

**Типографика:**
*   Основной шрифт страницы должен быть без засечек.
*   Размер основного текста должен быть задан в пикселях.
*   Цвет основного текста должен быть темно-серым.
*   Межстрочный интервал должен быть увеличен.
*   Все заголовки первого уровня должны быть написаны полностью заглавными буквами.
*   Заголовки первого уровня должны иметь увеличенное расстояние между буквами.
*   Заголовки второго уровня должны быть выровнены по центру.
*   Заголовки третьего уровня должны быть полужирными.
*   Первый абзац в описании выпуска должен иметь красную строку.
*   Текст в блоке длинной цитаты должен иметь увеличенное расстояние между словами.
*   Ссылки в навигационном меню должны иметь подчеркивание точками.
*   Цвет ссылок должен отличаться от цвета основного текста.

**Селекторы и Специфичность:**
*   Должна быть продемонстрирована работа специфичности: стиль, заданный через класс, должен переопределять стиль, заданный через тег, для одного и того же элемента.
*   Должен быть использован селектор потомка для стилизации элементов внутри навигации.
*   Должен быть использован селектор дочернего элемента первого уровня для стилизации непосредственных потомков в карточке ведущего.
*   Должен быть использован групповой селектор для задания одинакового размера шрифта подписям к медиа-контенту.

**Единицы измерения:**
*   Ширина основного контейнера контента должна быть задана в относительных единицах.
*   Размер шрифта для мелких подписей должен быть задан в `rem` или `em`.
