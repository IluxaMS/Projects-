### Разбор ошибок HTML (Оценка: 65/100)

#### 1. Критическая ошибка: Нарушение структуры списков в `nav`
*   **Как у вас:**
    ```html
    <nav>
        <li>...</li>
        <li>...</li>
    </nav>
    ```
*   **Ошибка:** Тег `<li>` (элемент списка) **не может** быть прямым потомком `<nav>`. Он обязан находиться внутри `<ul>` или `<ol>`. Это грубая ошибка валидации.
*   **Исправление:**
    ```html
    <nav>
        <ul>
            <li><a href="#...">...</a></li>
            ...
        </ul>
    </nav>
    ```

#### 2. Ошибка семантики: `figure` для карточек спикеров
*   **Как у вас:** Вы обернули всю карточку спикера (фото + имя + список + контакты) в тег `<figure>`.
*   **Ошибка:** `<figure>` предназначен для **автономного контента** (изображение, диаграмма, код), который имеет подпись (`figcaption`). Карточка спикера — это сложный составной блок, а не просто иллюстрация с подписью. Использовать `<figure>` как контейнер для всей карточки неверно.
*   **Исправление:** Используйте `<article>` или `<div>` с классом по БЭМ (например, `.speaker-card`). Внутри него уже можно использовать `<figure>` только для фото и имени, если нужно, но лучше просто `<img>` и текстовые блоки.

#### 3. Ошибка в таблице: `tfoot` без строки `tr`
*   **Как у вас:**
    ```html
    <tfoot>
        <td>Итог</td>
        <td>...</td>
    </tfoot>
    ```
*   **Ошибка:** Внутри `thead`, `tbody` и `tfoot` ячейки (`td`/`th`) должны обязательно находиться внутри строки `<tr>`.
*   **Исправление:**
    ```html
    <tfoot>
        <tr>
            <td>Итог</td>
            ...
        </tr>
    </tfoot>
    ```

#### 4. Ошибка в форме: Неправильный тип инпута
*   **ТЗ:** «Выбор времени звонка».
*   **Как у вас:** `<input type="datetime-local">`.
*   **Ошибка:** `datetime-local` выбирает дату и время. Для времени используется `type="time"`. В ТЗ также был отдельный пункт «Выбор даты и времени сеанса», где вы снова использовали `datetime-local` (это верно), но перепутали их местами или продублировали неправильно.
*   **Исправление:** Для времени звонка используйте `type="time"`.

#### 5. Ошибка в форме: `datalist`
*   **Как у вас:**
    ```html
    <input list="search-field">
    <datalist id="search-field">...</datalist>
    ```
*   **Комментарий:** Здесь все технически верно, но `input` должен иметь `type="search"` согласно ТЗ. У вас типа нет (по умолчанию text).

#### 6. Мелкие ошибки
*   Лишний пробел в ссылке: `href="#registration  "`.

### Разбор ошибок CSS (Оценка: 70/100)

#### 1. Критическая ошибка: `text-transform: capitalize` вместо `uppercase`
*   **ТЗ:** «Все заголовки первого уровня должны быть написаны **заглавными буквами**».
*   **Как у вас:** `text-transform: capitalize;`
*   **Ошибка:** `capitalize` делает Заглавной Только Первую Букву Каждого Слова. `uppercase` делает ВСЕ БУКВЫ ЗАГЛАВНЫМИ.
*   **Исправление:** `text-transform: uppercase;`

#### 2. Ошибка веса шрифта
*   **ТЗ:** «Заголовки третьего уровня должны иметь **полужирное** начертание».
*   **Как у вас:** `font-weight: 400;`
*   **Ошибка:** `400` — это нормальное (обычное) начертание. Полужирное — это `700` или `bold`.
*   **Исправление:** `font-weight: 700;` или `font-weight: bold;`

#### 3. Не выполнена задача по специфичности
*   **ТЗ:** «Должна быть продемонстрирована работа специфичности: стиль, заданный через класс, должен переопределять стиль, заданный через тег».
*   **Как у вас:** Вы задали стили для `a` (тег) и для `.description-of-the-event` (класс). Они не пересекаются. Вы не показали переопределение.
*   **Как надо было сделать:**
    ```css
    p { color: black; } /* Вес 1 */
    .my-text { color: red; } /* Вес 10, переопределяет черный для абзацев с этим классом */
    ```
    У вас есть попытка с `#id` и `.class` для `word-spacing`, но там нет конфликта "тег vs класс".

#### 4. Не использованы селекторы потомка и дочернего элемента
*   **ТЗ:** «Должен быть использован селектор потомка (пробел) и селектор дочернего элемента первого уровня (>)».
*   **Ваше оправдание:** "в HTML его их просто негде использовать".
*   **Вердикт:** Это неправда.
    *   *Селектор потомка:* `nav a` (ссылки внутри навигации), `section p` (абзацы внутри секции).
    *   *Селектор ребенка:* `.speaker-card > img` (картинка, лежащая непосредственно в карточке).
    *   Вы **не выполнили** этот пункт ТЗ.

#### 5. БЭМ нейминг
*   **Как у вас:** `.description-of-the-event__long-quote`
*   **Комментарий:** По БЭМ длинные имена блоков пишутся через дефис, но само название блока лучше делать короче. Однако, главная ошибка в том, что вы дали ID элементу, названному по БЭМ. ID должен быть уникальным и обычно не несет стилевой нагрузки в БЭМ, либо дублирует класс. Но допустимо.
