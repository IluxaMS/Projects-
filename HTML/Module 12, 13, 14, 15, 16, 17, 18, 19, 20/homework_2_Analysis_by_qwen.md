### Разбор ошибок HTML (Оценка: 75/100)

#### 1. Критическая ошибка: Тип кнопки отправки формы
*   **Как у вас:** `<button type="button" autofocus>Подписаться</button>`
*   **Ошибка:** Кнопка с типом `button` **не отправляет форму**. Она ничего не делает без JavaScript. Для отправки данных на сервер (или имитации отправки) необходимо использовать `type="submit"`. В ТЗ требовалась кнопка «Подписаться» (подразумевается действие отправки).
*   **Исправление:** `<button type="submit" autofocus>Подписаться</button>`

#### 2. Ошибка семантики: Тип поля Email
*   **Как у вас:** `<input type="text"> <!-- поле для ввода email -->`
*   **Ошибка:** Вы знаете тип `email`. Использование `text` для почты лишает браузер возможности проводить базовую валидацию (проверку наличия @).
*   **Исправление:** `<input type="email">`

#### 3. Ошибка структуры: Навигация и БЭМ
*   **Как у вас:**
    ```html
    <ul class="navigation-menu">
        <li class="navigation-menu">...</li>
    </ul>
    ```
*   **Ошибка:**
    1.  Вы присвоили один и тот же класс `navigation-menu` и списку `ul`, и элементам списка `li`. Это нарушает логику БЭМ и здравый смысл. Класс блока должен быть у родителя, у элементов — свои классы (например, `navigation-menu__item`).
    2.  В CSS вы обращаетесь к `nav .navigation-menu`. Так как класс висит на `ul`, селектор сработает, но стили применятся к `ul`, а не к ссылкам `a` внутри него. Ссылки останутся без подчеркивания точками, так как вы не обратились к тегу `a`.

#### 4. Ошибка в таблице: Комментарий vs Реальность
*   **Как у вас:**
    ```html
    <td colspan="2">Видео-трансляция студии</td> <!-- ячейка, объединяющая несколько строк -->
    ```
*   **Ошибка:** Атрибут `colspan` объединяет **столбцы** (columns). Ваш комментарий гласит «объединяющая несколько строк». Это ложь. `rowspan` — строки, `colspan` — столбцы.
    *   Ниже у вас есть `rowspan="2"` с комментарием «объединяющая несколько столбцов». Тоже ложь.
    *   Хотя технически атрибуты использованы верно, путаница в понимании недопустима.

#### 5. Лишний контент
*   **Как у вас:** Блок `.parent-1` в конце `body`.
*   **Комментарий:** ТЗ не просило создавать искусственные блоки для демонстрации CSS. Нужно было использовать существующую структуру (карточки ведущих). Это допустимо как "костыль", но снижает оценку за понимание контекста.

### Разбор ошибок CSS (Оценка: 70/100)

#### 1. Критическая ошибка: Селектор потомка не работает для ссылок
*   **ТЗ:** «Ссылки в навигационном меню должны иметь подчеркивание точками».
*   **Как у вас:**
    ```css
    nav .navigation-menu {
        text-decoration: underline dotted;
        color: rgb(165, 42, 42);
    }
    ```
*   **Ошибка:** Вы применили стили к `ul` (списку), а не к ссылкам `a` внутри него. Свойство `text-decoration` не наследуется ссылками автоматически таким образом, чтобы подчеркнуть текст ссылки, если оно задано родителю-списку. Ссылки остались стандартными (или унаследовали цвет, но не декорацию корректно).
*   **Исправление:** `nav a { text-decoration: underline dotted; color: ...; }`

#### 2. Не выполнена задача по селектору дочернего элемента (`>`)
*   **ТЗ:** «Использовать селектор дочернего элемента первого уровня для стилизации непосредственных потомков в карточке ведущего».
*   **Как у вас:** Вы создали отдельный блок `.parent-1` вне основного контента.
*   **Вердикт:** Это невыполнение условия задачи в контексте проекта. Вы должны были написать что-то вроде:
    ```css
    .presenter-card > h3 { font-weight: bold; }
    /* или */
    .presenter-card > figure { margin-bottom: 10px; }
    ```
    Использование "лабораторного" примера в конце файла показывает, что вы не смогли применить знание к реальной структуре, что является ошибкой.

#### 3. Специфичность
*   **Как у вас:**
    ```css
    .long-quote { word-spacing: 7px; }
    blockquote { word-spacing: 50px; }
    ```
*   **Комментарий:** Здесь все верно. Класс (вес 10) переопределяет тег (вес 1). Этот пункт засчитан.

#### 4. Единицы измерения
*   **ТЗ:** «Ширина основного контейнера контента должна быть задана в относительных единицах».
*   **Как у вас:** Вы не задавали ширину никакому контейнеру (так как не использовали `width` в CSS, только в обнулении).
*   **Вердикт:** Пункт не выполнен. Даже если вы не стилизовали ширину явно, можно было задать `main { width: 80%; }` или `section { width: 100vw; }`.
