### Практическое задание: «Лендинг архитектурного бюро "Monolith"»

Необходимо сверстать одностраничный сайт (лендинг) для архитектурного бюро.

#### 1. Структура и контент (HTML)

**Шапка сайта:**
*   Логотип или название бюро.
*   Навигационное меню с якорными ссылками, ведущими к разделам страницы.
*   Элемент взаимодействия (кнопка или ссылка), ведущий к форме обратной связи.

**Главный экран:**
*   Крупный заголовок с названием бюро и слоганом.
*   Краткое описание философии компании.
*   Визуальный акцент (изображение или графический элемент).

**Раздел «Наши проекты»:**
*   Заголовок раздела.
*   Три карточки проектов. Каждая карточка содержит:
    *   Изображение проекта.
    *   Название проекта.
    *   Краткое описание параметров (локация, площадь).
    *   Элемент действия (кнопка или ссылка).

**Раздел «Услуги»:**
*   Заголовок раздела.
*   Перечень услуг, оформленный в виде пар «Название услуги» — «Описание услуги».

**Раздел «О нас»:**
*   Заголовок раздела.
*   Текстовый блок с историей компании.
    *   В тексте должна присутствовать короткая цитата, встроенная в предложение.
    *   Должна присутствовать длинная оформленная цитата отдельным блоком.
    *   Одно ключевое слово должно быть визуально выделено.
*   Фотография с подписью.

**Раздел «Контакты и Форма»:**
*   Заголовок раздела.
*   Контактная информация: физический адрес, телефон, email.
*   Форма обратной связи, содержащая:
    *   Поле для ввода имени.
    *   Поле для ввода email.
    *   Поле для ввода телефона.
    *   Выпадающий список для выбора типа услуги.
    *   Многострочное поле для сообщения.
    *   Поле для подтверждения (любой подходящий тип input).
    *   Кнопка отправки формы.

**Подвал сайта:**
*   Информация об авторских правах.
*   Ссылки на внешние ресурсы.

---

#### 2. Стилизация (CSS)

**Шрифты:**
*   Подключите нестандартный шрифт для заголовков, используя правило `@font-face` или внешний источник.
*   Настройте параметры отображения шрифта при загрузке.
*   Для основного текста используйте стандартный шрифт без засечек.

**Блочная модель и Размеры:**
*   Примените модель расчета размеров, при которой внутренние отступы и границы включаются в общую ширину элемента.
*   Ограничьте максимальную ширину основного контента, чтобы он не растягивался на всю ширину больших экранов.
*   Центрируйте основной контент по горизонтали.
*   Задайте минимальную высоту для главного экрана, используя единицы измерения, зависящие от высоты окна браузера.
*   Используйте свойство управления переполнением контента для обрезки лишнего содержимого в одном из блоков.
*   Создайте блок с фиксированной высотой, внутри которого контент вызывает появление полосы прокрутки, если не помещается.

**Отображение и Видимость:**
*   Используйте различные значения свойства `display` для формирования структуры (блочные, строчно-блочные элементы).
*   Реализуйте скрытие одного из элементов так, чтобы он оставался в потоке документа (занимал место), но был невидим.
*   Реализуйте скрытие другого элемента так, чтобы он полностью исчезал из потока документа.

**Декоративное оформление:**
*   Оформите границы карточек проектов, используя разные стили линий.
*   Добавьте тень для карточек проектов.
*   Добавьте тень для текста главного заголовка.
*   Настройте уровень прозрачности для декоративных фоновых элементов.

**Специфичность и Селекторы:**
*   Переопределите стиль шрифта для одного конкретного заголовка, используя идентификатор, чтобы перебить стиль, заданный классом.
*   Используйте селектор дочернего элемента первого уровня для стилизации элементов списка услуг.
*   Используйте групповой селектор для задания одинаковых внешних отступов всем основным разделам страницы.

**Единицы измерения:**
*   Используйте относительные единицы измерения, зависящие от корневого элемента, для размеров шрифтов.
*   Используйте единицы измерения, зависящие от размеров viewport, для адаптивных блоков.
*   Используйте проценты для ширины внутренних элементов.

**Методология БЭМ:**
*   Именуйте все классы в соответствии с методологией БЭМ.
*   Используйте модификаторы для variations внешнего вида кнопок или карточек.
