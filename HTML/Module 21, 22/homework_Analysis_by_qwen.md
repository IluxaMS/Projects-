### Разбор ошибок HTML (Оценка: 92/100)

#### 1. Семантика раздела «Контакты» (-5 баллов)
*   **Как у вас:** Весь раздел с формой и контактами обернут в `<aside>`.
*   **Ошибка:** Тег `<aside>` предназначен для косвенного контента. Форма обратной связи и контакты на лендинге — это ключевой контент, требующий внимания пользователя.
*   **Исправление:** Замените `<aside id="contacts-and-form">` на `<section id="contacts-and-form">`.

#### 2. Мелкий недочет: `mailto` (-3 баллов)

*   **Ошибка:** В атрибуте `href` отсутствует адрес почты, ссылка не сработает корректно.

### Разбор ошибок CSS (Оценка: 80/100)

#### 1. Непонимание `box-sizing` (-5 баллов)
*   **ТЗ:** «Примените модель... при которой внутренние отступы и границы **включаются** в общую ширину».
*   **Как у вас:** `box-sizing: content-box;`
*   **Ошибка:** `content-box` — это стандарт, где padding/border добавляются сверху. Модель включения называется `border-box`.
*   **Исправление:** `box-sizing: border-box;`

#### 2. Ошибка в подключении шрифта (-5 баллов)
*   **Как у вас:** `font-family: "Antipasto Pro" sans-serif;` внутри `@font-face`.
*   **Ошибка:** В `@font-face` свойство `font-family` задает только имя шрифта. Fallback-шрифты (sans-serif) указываются только при использовании шрифта в селекторах (например, в `h1`).
*   **Исправление:**
    ```css
    @font-face { font-family: "Antipasto Pro"; ... }
    h1 { font-family: "Antipasto Pro", sans-serif; }
    ```

#### 3. Не выполнено ТЗ: Тень текста (-5 баллов)
*   **ТЗ:** «Добавьте тень для **текста** главного заголовка».
*   **Как у вас:** `.our-projects__title { box-shadow: ... }`
*   **Ошибка:**
    1.  Использован `box-shadow` (тень блока), а нужно `text-shadow` (тень текста).
    2.  Применено к `h2` проектов, а нужно к главному `h1`.
*   **Исправление:** `h1 { text-shadow: 2px 2px 4px #000; }`

#### 4. Не выполнено ТЗ: Селектор дочернего элемента (-5 баллов)
*   **ТЗ:** «Стилизовать элементы списка услуг».
*   **Как у вас:** `#services > dl { ... }`
*   **Ошибка:** Вы стилизовали контейнер `dl`. Нужно было стилизовать его непосредственных потомков (`dt` или `dd`).
*   **Исправление:** `#services > dl > dt { ... }`

#### 5. Задание на скрытие элементов (0 баллов штрафа)
*   **Комментарий:** Так как вы еще не изучали `display: none`, требование полностью убрать элемент из потока снимается.
*   **Ваше решение:** `visibility: hidden` и `opacity: 0`.
*   **Вердикт:** Оба свойства скрывают элемент визуально, но оставляют его в потоке (занимает место). Это соответствует части ТЗ («оставался в потоке»). Ошибкой не считается, задание адаптировано под ваш уровень.
