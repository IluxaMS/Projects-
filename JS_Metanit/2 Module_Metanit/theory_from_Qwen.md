# 1. Функции (Function Declaration)

Функция — это основной строительный блок логики в JS. Это именованный блок кода, который можно вызывать многократно.

**Синтаксис (Function Declaration):**
```javascript
function имяФункции(параметры) {
    // тело функции
    // инструкции
}
```

**Особенности:**
*   Объявление «всплывает» (Hoisting) — функцию можно вызвать *до* её объявления в коде.
*   Имеет собственное имя.

**Пример:**
```javascript
// Вызов до объявления возможен благодаря Hoisting
sayHello("Мой Господин"); 

function sayHello(name) {
    console.log(`Приветствую, ${name}!`);
}
```

# 2. Параметры функции и Аргументы

*   **Параметры** — это переменные, указанные в скобках при *объявлении* функции.
*   **Аргументы** — это конкретные значения, которые передаются в функцию при *вызове*.

### Виды параметров

1.  **Обязательные параметры:** Если аргумент не передан, параметр получает значение `undefined`.
2.  **Параметры по умолчанию (Default Parameters):** Позволяют задать значение, если аргумент не передан или равен `undefined`. (Появились в ES6).
3.  **Остаточные параметры (Rest Parameters):** Позволяют собрать все переданные аргументы в массив. Обозначаются троеточием `...`.

**Пример:**
```javascript
// name - обязательный, role - по умолчанию "Guest", ...args - остаточные
function createUser(name, role = "Guest", ...permissions) {
    console.log(`Пользователь: ${name}`);
    console.log(`Роль: ${role}`);
    console.log(`Права: ${permissions.join(", ")}`);
    
    // permissions внутри функции — это настоящий массив
    if (permissions.includes("admin")) {
        console.log("Доступ разрешен везде.");
    }
}

createUser("Alex"); 
// Вывод: Пользователь: Alex, Роль: Guest, Права: 

createUser("Ivan", "Admin", "read", "write", "delete");
// Вывод: Пользователь: Ivan, Роль: Admin, Права: read, write, delete
```

> **Важно:** Не путайте Rest parameters (`...args` в объявлении) с оператором Spread (`...array` при вызове), который «распаковывает» массив в отдельные аргументы.

# 3. Результат функции (Return)

Каждая функция в JavaScript всегда возвращает значение.
*   Если есть оператор `return`, возвращается указанное значение, и выполнение функции **немедленно прекращается**.
*   Если `return` отсутствует или написан просто `return;`, функция возвращает `undefined`.

**Особенности:**
*   Код после `return` никогда не выполнится («мертвый код»).
*   Функция может возвращать что угодно: число, строку, объект, другую функцию, `null`.

**Пример:**
```javascript
function calculatePower(base, exponent) {
    if (exponent < 0) {
        return "Ошибка: степень не может быть отрицательной"; // Возврат строки
    }
    
    const result = base ** exponent;
    return result; // Возврат числа
    
    console.log("Эта строка никогда не напечатается"); // Мертвый код
}

const power = calculatePower(2, 3); // 8
console.log(power * 2); // 16

// Функция без return
function logMessage(msg) {
    console.log(msg);
    // неявно возвращает undefined
}

const logged = logMessage("Тест");
console.log(logged); // undefined
```

# 4. Стрелочные функции (Arrow Functions)

Синтаксический сахар, появившийся в ES6. Делает код короче, но имеет важные отличия в поведении (особенно в контексте `this`, о чем ниже).

**Синтаксис:**
```javascript
const имя = (параметры) => выражение;
```

**Правила сокращения:**
1.  Если один параметр, скобки `()` можно опустить.
2.  Если тело функции состоит из одного выражения, фигурные скобки `{}` и слово `return` можно опустить (возврат происходит автоматически).
3.  Если параметров нет, нужны пустые скобки `()`.

**Примеры:**
```javascript
// Обычная функция
function sum(a, b) {
    return a + b;
}

// Стрелочная (полная форма)
const sumArrow = (a, b) => {
    return a + b;
};

// Стрелочная (сокращенная форма - implicit return)
const sumShort = (a, b) => a + b;

// Один параметр (скобки не нужны)
const double = n => n * 2;

// Нет параметров
const sayHi = () => console.log("Привет!");

// Возврат объекта (нужны скобки, иначе JS подумает, что это тело функции)
const getUser = () => ({ name: "Admin", id: 1 });
```

**Критическое отличие:** Стрелочные функции **не имеют своего собственного `this`**. Они берут `this` из окружающего лексического контекста (откуда были вызваны). Поэтому их нельзя использовать как методы объектов, если нужен доступ к самому объекту через `this`, и нельзя использовать как конструкторы (через `new`).

# 5. Область видимости переменных (Scope)

Область видимости определяет, где в коде доступна переменная.

1.  **Глобальная область:** Переменные объявлены вне любых функций. Доступны везде.
2.  **Функциональная область (Function Scope):** Переменные, объявленные через `var` внутри функции, видны только внутри этой функции.
3.  **Блочная область (Block Scope):** Переменные, объявленные через `let` и `const` внутри блока `{...}` (if, for, просто блок), видны только внутри этого блока.

**Пример:**
```javascript
let globalVar = "Я глобальная";

function testScope() {
    let funcVar = "Я видна только в функции";
    
    if (true) {
        let blockVar = "Я видна только в этом if";
        var oldVar = "Я всплыву на уровень функции (var)";
        console.log(blockVar); // OK
    }
    
    console.log(oldVar); // OK (var игнорирует блоки)
    // console.log(blockVar); // Ошибка: blockVar is not defined
}

testScope();
// console.log(funcVar); // Ошибка
// console.log(blockVar); // Ошибка
```

# 6. Замыкания (Closures)

Это одна из самых мощных и сложных концепций JavaScript.

**Определение:** Замыкание — это способность функции запоминать и иметь доступ к переменным из своей внешней (лексической) области видимости, даже когда эта внешняя функция уже завершила выполнение.

Функция «замыкает» в себе окружение, в котором была создана.

**Пример 1 (Счетчик):**
```javascript
function createCounter() {
    let count = 0; // Локальная переменная внешней функции
    
    // Внутренняя функция замыкает переменную count
    return function() {
        count++;
        return count;
    };
}

const counter1 = createCounter();
const counter2 = createCounter();

console.log(counter1()); // 1
console.log(counter1()); // 2
console.log(counter1()); // 3

console.log(counter2()); // 1 (У каждого счетчика свое замыкание и свой count)
```
Переменная `count` не уничтожается после выполнения `createCounter`, потому что на неё ссылается внутренняя функция.

**Пример 2 (Фабрика приветствий):**
```javascript
function makeGreeter(greeting) {
    return function(name) {
        console.log(`${greeting}, ${name}!`);
    };
}

const sayHello = makeGreeter("Привет");
const sayGoodbye = makeGreeter("Пока");

sayHello("Мой Господин"); // Привет, Мой Господин!
sayGoodbye("Враг");       // Пока, Враг!
```

# 7. Функции IIFE (Immediately Invoked Function Expression)

Это функция, которая определяется и сразу же выполняется. Использовалась до появления `let/const` для создания изолированной области видимости и предотвращения загрязнения глобального пространства имен.

**Синтаксис:**
```javascript
(function() {
    // Код выполняется сразу
    let secret = "Тайна";
    console.log("IIFE запущена");
})(); // Обратите внимание на скобки () в конце
```

Или со стрелочной функцией:
```javascript
(() => {
    console.log("Стрелочная IIFE");
})();
```

**Зачем нужно сейчас?**
В современном коде используется реже из-за блочной области видимости (`{ let x = 1; }`), но всё еще применяется в модульных паттернах или для инкапсуляции логики в библиотеках.

# 8. Рекурсивные функции

Рекурсия — это когда функция вызывает саму себя. Обязательно требуется **условие выхода (базовый случай)**, иначе возникнет бесконечный цикл и переполнение стека (Stack Overflow).

**Пример 1 (Факториал):**
$5! = 5 \times 4 \times 3 \times 2 \times 1$
```javascript
function factorial(n) {
    // Базовый случай
    if (n === 1 || n === 0) {
        return 1;
    }
    // Рекурсивный шаг
    return n * factorial(n - 1);
}

console.log(factorial(5)); // 120
/*
Ход выполнения:
factorial(5) -> 5 * factorial(4)
factorial(4) -> 4 * factorial(3)
factorial(3) -> 3 * factorial(2)
factorial(2) -> 2 * factorial(1)
factorial(1) -> 1 (возврат)
... сборка результата обратно ...
*/
```

**Пример 2 (Обход вложенных структур):**
```javascript
const data = {
    name: "Root",
    children: [
        { name: "Child 1" },
        { 
            name: "Child 2", 
            children: [{ name: "Grandchild" }] 
        }
    ]
};

function printTree(node, level = 0) {
    console.log("-".repeat(level) + node.name);
    if (node.children) {
        node.children.forEach(child => printTree(child, level + 1));
    }
}

printTree(data);
```

# 9. Переопределение функций

В JavaScript функции — это объекты первого класса. Их можно присваивать переменным, передавать как аргументы и... перезаписывать.

**Сценарий:**
```javascript
function log(msg) {
    console.log("LOG: " + msg);
}

log("Сообщение 1"); // LOG: Сообщение 1

// Переопределяем функцию
log = function(msg) {
    alert("ALERT: " + msg);
};

log("Сообщение 2"); // Появится всплывающее окно (в браузере)
```

**Полезное применение (Полифилы и Моки):**
Часто используется в тестировании, чтобы подменить реальную функцию (например, запрос к серверу) на фейковую, которая возвращает заранее известные данные.

```javascript
// Сохраняем оригинал
const originalFetch = window.fetch;

// Подменяем для теста
window.fetch = function() {
    return Promise.resolve({ json: () => ({ id: 1, name: "Test" }) });
};

// ... запускаем тесты ...

// Возвращаем как было
window.fetch = originalFetch;
```

# 10. Hoisting (Поднятие)

Механизм, при котором объявления переменных и функций «поднимаются» в верхнюю часть своей области видимости перед выполнением кода.

**Как это работает:**
1.  **Function Declaration:** Поднимается полностью (и объявление, и тело). Можно вызывать до объявления.
2.  **Variable (`var`):** Поднимается объявление, но инициализация остается на месте. Значение до строки объявления — `undefined`.
3.  **Variable (`let`, `const`):** Поднимается объявление, но попадает в «временную мертвую зону» (TDZ). Любое обращение до строки объявления вызовет `ReferenceError`.

**Пример:**
```javascript
console.log(funcDecl); // [Function: funcDecl] - Работает!
funcDecl(); 

function funcDecl() { console.log("Declaration"); }

console.log(varVar); // undefined - Объявление поднято, значение нет
var varVar = 10;

console.log(letVar); // Ошибка: Cannot access 'letVar' before initialization
let letVar = 20;
```

> **Совет:** Всегда объявляйте переменные и функции перед их использованием, чтобы избежать путаницы с Hoisting.

# 11. Передача параметров: По значению vs По ссылке

Это фундаментальное понимание того, как JS работает с памятью.

### Примитивные типы (По значению)
`Number`, `String`, `Boolean`, `Null`, `Undefined`, `Symbol`, `BigInt`.
При передаче в функцию создается **копия** значения. Изменение параметра внутри функции **не влияет** на исходную переменную.

```javascript
let a = 10;

function changeValue(num) {
    num = 100;
    console.log("Внутри:", num); // 100
}

changeValue(a);
console.log("Снаружи:", a); // 10 (не изменилось)
```

### Сложные типы (По ссылке / Copy of Reference)
`Object`, `Array`, `Function`.
Переменная хранит не сам объект, а **ссылку** (адрес в памяти) на него. При передаче в функцию копируется **ссылка**, а не сам объект. Обе переменные (снаружи и внутри) указывают на **один и тот же объект**.
*   Изменение **свойств** объекта внутри функции отразится снаружи.
*   Но если **переприсвоить** сам параметр (указать на новый объект), это не затронет внешнюю переменную.

```javascript
let user = { name: "Admin", score: 0 };

function changeObject(obj) {
    // Изменение свойства (влияет на оригинал!)
    obj.score = 100;
    obj.name = "SuperAdmin";
    i
    // Переприсвоение ссылки (НЕ влияет на оригинал!)
    obj = { name: "Hacker", score: 999 }; 
    console.log("Внутри (после переприсвоения):", obj); // { name: "Hacker"... }
}

changeObject(user);

console.log("Снаружи:", user); 
// { name: "SuperAdmin", score: 100 } 
// Имя и счет изменились, но объект не стал "Hacker"
```

**Как передать объект «по значению» (чтобы не менять оригинал)?**
Нужно создать копию перед передачей или внутри функции:
```javascript
// shallow copy (поверхностная копия)
changeObject({ ...user }); 
// или
changeObject(Object.assign({}, user));
// или (для глубокой копии, если нет вложенных объектов)
changeObject(JSON.parse(JSON.stringify(user)));
```
