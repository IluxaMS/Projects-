### Задание 4: «Библиотека утилит для работы с коллекциями (Call/Apply)»

**Контекст:**
Вы пишете библиотеку вспомогательных функций. У вас есть обычные функции, которые должны работать с объектами, но вы хотите вызывать их в контексте разных объектов, используя `call` и `apply`. Также вам нужно создать фабрику объектов с особым поведением.

**Входные данные:**
*   Объект `teamA`: `{ name: "Alpha", members: ["John", "Jane"], score: 10 }`.
*   Объект `teamB`: `{ name: "Beta", members: ["Bob"], score: 5 }`.

**Требования к логике (пошагово):**
1.  **Функция как объект:**
    *   Создайте функцию `addMember(newMember)`, которая добавляет элемент в массив `this.members` текущего объекта и увеличивает `this.score` на 1.
    *   Эта функция не должна быть методом какого-то конкретного объекта изначально.
2.  **Использование call/apply:**
    *   Вызовите `addMember` в контексте `teamA`, передав аргумент `"Alice"` через `call`.
    *   Вызовите `addMember` в контексте `teamB`, передав аргументы `["Charlie", "Dave"]` (несколько человек) через `apply`. *Подсказка:* Вам придется адаптировать функцию или вызвать её несколько раз в цикле внутри apply-обертки, либо сделать функцию принимающую rest-параметры. Давайте сделаем функцию принимающей rest: `function addMembers(...newMembers)`.
3.  **Фабрика объектов через Object.create:**
    *   Создайте объект-прототип `teamPrototype` с методом `getInfo()`, возвращающим строку `"Команда [Name]: [Count] участников, счет [Score]"`.
    *   Напишите функцию `createTeam(name)`, которая создает новый объект на основе `teamPrototype` через `Object.create`.
    *   Инициализируйте свойства `name`, `members` (пустой массив), `score` (0) в новом объекте.
    *   Верните этот объект.
4.  **Перебор и проверка:**
    *   Создайте 3 команды через фабрику.
    *   Добавьте им участников (используя `call` с вашей функцией из п.1).
    *   Пройдитесь циклом по всем созданным командам (храните их в массиве).
    *   Для каждой команды вызвите `getInfo()`.
    *   Проверьте наличие свойства `captain` у каждой команды через `in`. Если нет — добавьте программно.
5.  **Деструктуризация с переименованием:**
    *   Напишите функцию `showScore({ score: points, name: teamName })`, которая деструктуризирует объект, переименовывая поля.
    *   Выведите: `"Счет команды [teamName] равен [points]"`.

**Ожидаемый результат:**
Функции, работающие в динамическом контексте (`this` меняется через call/apply), фабрика объектов с наследованием от прототипа, перебор коллекций объектов, продвинутая деструктуризация.

*Цель: `call`/`apply`, функции как объекты, `Object.create` для фабрик, перебор, деструктуризация с алиасами.*
