### Задание 3: «Умный конфигуратор с инкапсуляцией и геттерами»

**Контекст:**
Вам нужно создать объект конфигурации приложения, который защищает свои критические данные от прямого изменения, использует геттеры/сеттеры для валидации и позволяет создавать свои копии с расширенными возможностями.

**Требования к логике (пошагово):**
1.  **Создание объекта с аксессорами:**
    *   Создайте объект `appConfig`.
    *   Используйте внутреннее свойство `_version` (скрытое соглашение).
    *   Реализуйте **геттер** `version`, который возвращает строку вида `"v" + _version`.
    *   Реализуйте **сеттер** `version`, который принимает число. Если число меньше 1, выбрасывает ошибку или устанавливает 1. Иначе обновляет `_version`.
2.  **Конфигурация свойств:**
    *   Добавьте свойство `apiKey` через `Object.defineProperty`.
    *   Сделайте его `writable: false` (нельзя перезаписать), `enumerable: false` (не видно в цикле), `configurable: false` (нельзя удалить).
    *   Присвойте ему значение `"SECRET_123"`.
3.  **Деструктуризация:**
    *   Напишите функцию `printConfig({ version, apiKey })`, которая принимает объект и сразу деструктуризирует эти два свойства.
    *   Вызовите эту функцию, передав `appConfig`. Обратите внимание: `apiKey` не виден при обычном переборе, но доступен по имени.
4.  **Расширение объекта (Прототипы):**
    *   Создайте новый объект `extendedConfig`, который наследует все свойства `appConfig`.
    *   Используйте `Object.create(appConfig)`.
    *   Добавьте в `extendedConfig` новое свойство `debugMode: true`.
    *   Проверьте, что при обращении к `extendedConfig.version` срабатывает геттер из родителя (`appConfig`).
5.  **Защита объекта:**
    *   Попробуйте изменить `appConfig.apiKey` напрямую. Убедитесь, что это не удалось (в строгом режиме будет ошибка, в обычном — игнор).
    *   Попробуйте удалить `appConfig._version`. Если свойство настраивалось через defineProperty без явного указания configurable (по умолчанию true для простых присваиваний, но лучше явно запретить), проверьте поведение. *Уточнение:* Сделайте `_version` тоже через `defineProperty` с `configurable: false` для полной защиты.
    *   Примените `Object.freeze(appConfig)` в конце. Попробуйте изменить `version` через сеттер (если freeze блокирует и сеттеры — проверьте поведение, обычно freeze делает свойства read-only, и сеттеры перестают работать или вызывают ошибку).

**Ожидаемый результат:**
Объект с защищенными свойствами, работающими геттерами/сеттерами, успешная деструктуризация, создание наследника через `Object.create`, демонстрация невозможности изменения замороженных/защищенных свойств.

*Цель: Геттеры/сеттеры, `Object.defineProperty`, `Object.freeze`, `Object.create`, деструктуризация, инкапсуляция.*
