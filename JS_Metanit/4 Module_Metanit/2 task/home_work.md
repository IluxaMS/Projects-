### Задание 2: «Конструктор персонажей RPG игры»

**Контекст:**
Вы разрабатываете движок для простой RPG игры. Вам нужно создать функцию-конструктор для генерации персонажей, настроить им методы через прототип (чтобы не дублировать код) и реализовать наследование для особого типа персонажей.

**Требования к логике (пошагово):**
1.  **Функция-конструктор:**
    *   Создайте функцию `Character(name, health, power)`.
    *   Внутри неё присвойте свойства `name`, `health`, `power` контексту `this`.
    *   Инициализируйте свойство `isAlive` значением `true`.
2.  **Методы в прототипе:**
    *   Добавьте метод `attack(target)` в `Character.prototype`.
    *   Логика метода: уменьшает `health` цели на значение `this.power`. Выводит сообщение: `"[Имя атакующего] атакует [Имя цели] и наносит [Урон] урона"`.
    *   Если `health` цели становится <= 0, установите `target.isAlive = false` и выведите сообщение о смерти.
    *   **Важно:** Метод должен использовать `this` для доступа к силам атакующего.
3.  **Наследование конструкторов:**
    *   Создайте конструктор `Mage(name, health, power, mana)`.
    *   Внутри него вызовите конструктор `Character` через `call` или `apply`, передав `this` и аргументы (name, health, power), чтобы инициализировать базовые свойства.
    *   Добавьте свое свойство `mana`.
4.  **Настройка прототипа наследника:**
    *   Настройте `Mage.prototype` так, чтобы он наследовал от `Character.prototype`. Используйте `Object.create`.
    *   Не забудьте восстановить свойство `constructor` у `Mage.prototype`.
5.  **Уникальный метод мага:**
    *   Добавьте в `Mage.prototype` метод `heal(amount)`.
    *   Логика: если `mana >= amount`, уменьшает ману и увеличивает здоровье (не больше максимума, например, 100). Иначе выводит "Недостаточно маны".
6.  **Тестирование:**
    *   Создайте воина (`new Character(...)`) и мага (`new Mage(...)`).
    *   Заставьте мага атаковать воина.
    *   Заставьте воина атаковать мага.
    *   Попробуйте вызвать метод `heal` у воина (должно быть `undefined` или ошибка, так как у воина его нет).

**Ожидаемый результат:**
Работающая система с наследованием. Методы `attack` общие (лежат в прототипе), `heal` только у магов. Свойства инициализируются корректно через `call`.

*Цель: Функции-конструкторы, `this`, прототипы (`prototype`), наследование через `Object.create`, методы `call`/`apply`.*
