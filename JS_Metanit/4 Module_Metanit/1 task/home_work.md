### Задание 1: «Учет личных финансов с вложенной структурой»

**Контекст:**
Вам нужно создать систему учета личных финансов пользователя. Данные хранятся в сложном вложенном объекте. Ваша задача — научиться безопасно извлекать данные, проверять их наличие и правильно копировать структуру, не ломая оригинал.

**Входные данные:**
Объект `userFinance`, содержащий:
*   `name` (строка): Имя владельца.
*   `balance` (число): Текущий баланс.
*   `cards` (массив объектов): Список карт. Каждая карта имеет `number` (строка), `type` (строка: "credit" или "debit"), `limit` (число, может отсутствовать).
*   `settings` (объект): Настройки `{ currency: "USD", notifications: true }`.

Пример:
```javascript
const userFinance = {
    name: "Alex",
    balance: 5000,
    cards: [
        { number: "1234", type: "debit", limit: 0 },
        { number: "5678", type: "credit", limit: 10000 }
    ],
    settings: { currency: "USD", notifications: true }
};
```

**Требования к логике (пошагово):**
1.  **Безопасный доступ:**
    *   Напишите код, который пытается получить значение свойства `limit` у второй карты в массиве.
    *   Используйте **оператор опциональной цепочки (`?.`)**, чтобы код не выдавал ошибку, если массив `cards` пуст или вторая карта отсутствует.
    *   Если лимита нет (свойство отсутствует или равно 0), выведите сообщение "Лимит не установлен".
2.  **Проверка наличия свойств:**
    *   Проверьте, есть ли в объекте `settings` свойство `theme`. Используйте оператор `in` или метод `hasOwnProperty` (или `Object.hasOwn`).
    *   Если свойства нет, добавьте его со значением `"dark"`.
3.  **Перебор свойств:**
    *   Пройдитесь циклом по всем собственным ключам объекта `userFinance` (используйте `Object.keys` или `for...in` с проверкой).
    *   Выведите в консоль список ключей, исключая вложенные объекты и массивы (проверьте тип значения через `typeof`).
4.  **Копирование объекта:**
    *   Создайте копию объекта `userFinance`.
    *   **Важно:** Сделайте **глубокую копию** (deep copy) хотя бы для уровня `cards`, чтобы изменение типа карты в копии не меняло оригинал. Можно использовать комбинацию spread-оператора для верхнего уровня и `.map()` с spread для массива карт, либо `JSON.parse(JSON.stringify(...))`, если помните про его ограничения (но в данном случае данные простые).
    *   Измените в копии тип первой карты на "gold".
    *   Убедитесь и выведите в консоль, что в оригинальном объекте тип карты остался прежним (демонстрация того, что поверхностное копирование не сработало бы здесь корректно без дополнительных усилий).
5.  **Сравнение:**
    *   Попробуйте сравнить оригинал и копию через `==` или `===`. Выведите результат (должно быть `false`, так как это разные ссылки).

**Ожидаемый результат:**
Безопасное извлечение данных, добавление отсутствующего свойства, список ключей верхнего уровня, две независимые структуры данных (оригинал и копия), подтверждение, что они разные.

*Цель: Отработать `?.`, проверку свойств (`in`/`hasOwn`), перебор (`Object.keys`), глубокое копирование и сравнение по ссылке.*
