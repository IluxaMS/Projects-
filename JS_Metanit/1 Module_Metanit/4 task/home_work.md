### Задание 4: «Генератор Отчета по Продажам с Валидацией»
Представьте, что вы получаете сырые данные о ежедневных продажах в виде массива строк и чисел вперемешку (например: `["100", 250, "invalid", "", 0, "300", null, 50]`). Некоторые записи могут быть пустыми строками, некоторые — текстом, некоторые — нулями.

**Требования к логике:**
1.  Напишите функцию (или блок кода), которая принимает этот сырой массив.
2.  Внутри цикла обработайте каждый элемент:
    *   Попытайтесь преобразовать элемент в число.
    *   Если результат преобразования `NaN`, `null` или пустая строка после тримминга — считайте продажу равной 0, но зафиксируйте факт ошибки в отдельный массив ошибок (храните индекс и исходное значение).
    *   Если число отрицательное (возврат товара), учтите это в общей сумме, но пометьте флаг "Есть возвраты".
3.  Рассчитайте общую сумму продаж, количество успешных транзакций (где число > 0) и среднее значение чека.
4.  Используйте тернарный оператор для формирования итоговой строки статуса: если есть возвраты, добавить пометку "(Внимание: были возвраты)", иначе "(Все чисто)".
5.  Реализуйте логику бонуса: если общая сумма больше 1000, добавить бонус 5% к итогу, иначе бонус 0.
6.  В конце выведите подробный отчет в консоль, используя шаблонные строки, включая список найденных ошибок.
7.  **Задача по отладке:** Специально оставьте в коде одну логическую ошибку (например, неверное условие сравнения или ошибку в индексе цикла), запустите код с открытым отладчиком, найдите проблему через точки останова (breakpoints) и стек вызовов, затем исправьте её.

*Цель: глубокое преобразование типов, сложная валидация данных в цикле, агрегация данных, использование отладчика.*
