# 1. Переменные и Константы

В современном JavaScript (стандарт ES6+) существует три способа объявления переменных: `let`, `const` и устаревший `var`.

### Основные правила
*   **`let`**: Объявляет переменную с блочной областью видимости. Значение можно изменять (переприсваивать).
*   **`const`**: Объявляет константу с блочной областью видимости. Значение нельзя изменить после присваивания (ссылка неизменна). *Важно: если объект или массив объявлены через `const`, менять их содержимое можно, но нельзя присвоить им новый объект.*
*   **`var`**: Устаревший способ. Имеет функциональную область видимости и подвержен "поднятию" (hoisting), что часто приводит к ошибкам. **Рекомендуется не использовать.**

### Примеры

```javascript
// Использование let (изменяемая переменная)
let score = 0;
score = 10; // Valid: значение изменено
console.log(score); // 10

// Использование const (неизменяемая ссылка)
const PI = 3.14159;
// PI = 3.14; // Ошибка: Assignment to constant variable.

// Нюанс с объектами и массивами в const
const user = { name: "Admin", role: "SuperUser" };
user.role = "GodMode"; // Valid: мы меняем свойство объекта, а не ссылку
console.log(user.role); // "GodMode"

// user = {}; // Ошибка: попытка переприсвоить ссылку

// Область видимости (Block Scope)
if (true) {
    let blockVar = "Я вижу свет";
    const blockConst = "Я тоже";
    var globalVar = "Я везде"; // var игнорирует блоки {}
}
// console.log(blockVar); // Ошибка: blockVar is not defined
console.log(globalVar); // "Я везде" (var всплыл наружу)
```

# 2. Типы данных

JavaScript — язык с динамической типизацией. Тип определяется в момент выполнения. Существует 7 примитивных типов и один сложный (объект).

### Примитивные типы
1.  **String**: Строки.
2.  **Number**: Числа (целые и дробные, включая `Infinity`, `-Infinity`, `NaN`).
3.  **BigInt**: Целые числа произвольной длины (суффикс `n`).
4.  **Boolean**: Логический тип (`true`, `false`).
5.  **Null**: Специальное значение, означающее "пусто" или "значение неизвестно".
6.  **Undefined**: Значение переменной, которая была объявлена, но не инициализирована.
7.  **Symbol**: Уникальный идентификатор (используется для скрытых свойств объектов).

### Сложный тип
*   **Object**: Коллекция ключ-значение (включая массивы, функции, даты).

### Примеры и проверка типов

```javascript
const str = "Привет, Господин";
const num = 42;
const bigNum = 9007199254740991n; // BigInt
const isActive = true;
const empty = null;
let notDefined; // undefined
const uniqueId = Symbol("id");

const person = { name: "Вы" }; // Object
const arr = [1, 2, 3]; // Тоже Object (массив)

// Оператор typeof
console.log(typeof str);       // "string"
console.log(typeof num);       // "number"
console.log(typeof bigNum);    // "bigint"
console.log(typeof isActive);  // "boolean"
console.log(typeof empty);     // "object" (историческая ошибка JS, null считается объектом)
console.log(typeof notDefined);// "undefined"
console.log(typeof arr);       // "object" (для проверки массива используют Array.isArray(arr))
```

# 3. Арифметические операции

Стандартный набор математических операторов.

*   `+` (Сложение): Для чисел складывает, для строк — конкатенирует (объединяет).
*   `-` (Вычитание), `*` (Умножение), `/` (Деление).
*   `%` (Остаток от деления).
*   `**` (Возведение в степень).
*   `++` (Инкремент), `--` (Декремент).

### Особенности и примеры

```javascript
let a = 10;
let b = 3;

console.log(a + b);   // 13
console.log(a - b);   // 7
console.log(a * b);   // 30
console.log(a / b);   // 3.3333333333333335
console.log(a % b);   // 1 (остаток от 10 / 3)
console.log(a ** b);  // 1000 (10 в степени 3)

// Инкремент
let x = 5;
x++; // x стало 6
++x; // x стало 7

// Конкатенация строк vs Сложение
console.log("5" + 3); // "53" (строка + число = строка)
console.log("5" - 3); // 2 (строка приводится к числу при вычитании)
console.log("5" * "2"); // 10
```

# 4. Поразрядные (Битовые) операции

Эти операции работают непосредственно с двоичным представлением чисел (32-битные целые со знаком). Используются редко, в основном для оптимизации или работы с флагами/масками.

*   `&` (AND): Бит становится 1, только если оба соответствующих бита равны 1.
*   `|` (OR): Бит становится 1, если хотя бы один из битов равен 1.
*   `^` (XOR): Бит становится 1, если биты различны.
*   `~` (NOT): Инвертирует все биты.
*   `<<` (Сдвиг влево), `>>` (Сдвиг вправо с сохранением знака), `>>>` (Сдвиг вправо без знака).

### Пример (Флаги доступа)
Представим права доступа как биты: `1` (чтение), `2` (запись), `4` (удаление).

```javascript
const READ = 1;  // 001
const WRITE = 2; // 010
const DELETE = 4;// 100

// Пользователь имеет права на чтение и запись
let userPermissions = READ | WRITE; // 001 | 010 = 011 (число 3)

// Проверка: есть ли право на запись?
if (userPermissions & WRITE) {
    console.log("Запись разрешена"); // Выполнится
}

// Проверка: есть ли право на удаление?
if (userPermissions & DELETE) {
    console.log("Удаление разрешено");
} else {
    console.log("Удаление запрещено"); // Выполнится
}

// Добавление права на удаление
userPermissions |= DELETE; // 011 | 100 = 111 (число 7)

// Сдвиг влево (умножение на 2 в степени сдвига)
console.log(1 << 3); // 8 (1 * 2^3)
```

# 5. Условные выражения и Операторы `?:` и `??`

### Тернарный оператор `?:`
Краткая форма записи `if...else`.
Синтаксис: `условие ? выражение_если_истина : выражение_если_ложь`.

```javascript
const age = 20;
const status = age >= 18 ? "Совершеннолетний" : "Несовершеннолетний";
console.log(status); // "Совершеннолетний"

// Вложенный тернарный (не рекомендуется делать глубоким для читаемости)
const level = 10;
const rank = level > 20 ? "Master" : (level > 5 ? "Advanced" : "Beginner");
```

### Оператор нулевого слияния `??` (Nullish Coalescing)
Возвращает первый операнд, если он **не** `null` и **не** `undefined`. В отличие от логического ИЛИ (`||`), он не считает ложными `0`, `""` (пустую строку) или `false`.

```javascript
const defaultValue = "Гость";

// Сценарий с пустым именем (строка), но пользователь существует
let userName = ""; 

// Плохо: || вернет default, так как "" это falsy
console.log(userName || defaultValue); // "Гость" (Ошибка логики, имя было пустой строкой, а не отсутствием)

// Хорошо: ?? проверит только на null/undefined
console.log(userName ?? defaultValue); // "" (Пустая строка сохранена, так как она не null)

// Пример с нулем
let count = 0;
console.log(count || 10); // 10 (0 считается ложью)
console.log(count ?? 10); // 0 (0 является валидным значением)
```

# 6. Преобразования данных (Type Conversion)

JS выполняет преобразования автоматически (неявное) или явно по запросу разработчика.

### Неявное приведение (Coercion)
Происходит при операциях между разными типами.
```javascript
console.log("5" - 1);   // 4 (строка стала числом)
console.log("5" + 1);   // "51" (число стало строкой)
console.log(true + 1);  // 2 (true стало 1)
console.log(false + 1); // 1
```

### Явное приведение
Используется, когда нужно гарантировать тип.

```javascript
// В число
Number("123");      // 123
Number("abc");      // NaN
parseInt("100px");  // 100 (извлекает целое число из начала строки)
parseFloat("3.14m");// 3.14
+"42";              // 42 (унарный плюс)

// В строку
String(123);        // "123"
(123).toString();   // "123"

// В булево
Boolean("");        // false (пустая строка)
Boolean("0");       // true (непустая строка)
Boolean(0);         // false
Boolean(null);      // false
Boolean(undefined); // false
Boolean([]);        // true (пустой массив - это объект, он truthy)
```

# 7. Введение в массивы

Массивы — это упорядоченные коллекции данных. В JS они являются объектами специального типа.

### Создание и доступ
```javascript
const fruits = ["Яблоко", "Банан", "Вишня"];
const mixed = [1, "два", true, null, { key: "value" }];

console.log(fruits[0]); // "Яблоко" (индексация с 0)
console.log(fruits.length); // 3
fruits[1] = "Груша"; // Изменение элемента
```

### Основные методы
```javascript
const nums = [1, 2, 3];

// Добавление/Удаление
nums.push(4);       // [1, 2, 3, 4] (в конец)
nums.pop();         // удаляет 4, возвращает его
nums.unshift(0);    // [0, 1, 2, 3] (в начало)
nums.shift();       // удаляет 0

// Работа с содержимым
const joined = nums.join(", "); // "1, 2, 3" (массив -> строка)
const sliced = nums.slice(1, 3); // [2, 3] (копия части массива, оригинал не меняется)
nums.splice(1, 1, "NEW"); // Удаляет 1 элемент с индекса 1 и вставляет "NEW"

// Перебор (современный подход)
nums.forEach((item, index) => {
    console.log(`Индекс ${index}: ${item}`);
});

// Трансформация (возвращает новый массив)
const doubled = nums.map(n => n * 2);

// Фильтрация
const evens = nums.filter(n => n % 2 === 0);
```

# 8. Условные конструкции

Позволяют выполнять код только при выполнении определенных условий.

### `if ... else if ... else`
```javascript
const temperature = 25;

if (temperature < 0) {
    console.log("Ледяной холод");
} else if (temperature < 15) {
    console.log("Прохладно");
} else if (temperature < 25) {
    console.log("Тепло");
} else {
    console.log("Жарко");
}
```

### `switch`
Используется для сравнения одного значения с множеством вариантов. Важно не забывать `break`, иначе выполнится следующий кейс ("проваливание").

```javascript
const day = 3;
let dayName;

switch (day) {
    case 1:
        dayName = "Понедельник";
        break;
    case 2:
        dayName = "Вторник";
        break;
    case 3:
    case 4: 
        // Группировка случаев: для 3 и 4 будет одно действие
        dayName = "Середина недели";
        break;
    default:
        dayName = "Неизвестный день";
}
console.log(dayName); // "Середина недели"
```

# 9. Циклы

Используются для многократного выполнения кода.

### `for` (Классический)
Когда известно количество итераций или нужен контроль индекса.
```javascript
for (let i = 0; i < 5; i++) {
    console.log(`Итерация ${i}`);
}
```

### `while` и `do...while`
Выполняются, пока условие истинно. `do...while` гарантирует выполнение тела цикла хотя бы один раз.
```javascript
let count = 3;
while (count > 0) {
    console.log(count);
    count--;
}

// do...while
let input;
do {
    // input = prompt("Введите 'stop' для выхода");
    input = "stop"; // эмуляция
} while (input !== "stop");
```

### `for...of` (Перебор итерируемых объектов)
Самый удобный способ перебора массивов, строк, Map, Set.
```javascript
const colors = ["red", "green", "blue"];
for (const color of colors) {
    console.log(color); // red, green, blue
}
```

### `for...in` (Перебор ключей объекта)
**Не рекомендуется** для массивов (порядок не гарантирован, перебирает ключи как строки). Идеален для объектов.
```javascript
const user = { name: "Alex", age: 30 };
for (const key in user) {
    console.log(`${key}: ${user[key]}`);
}
```

### Управление циклом
*   `break`: Полностью выходит из цикла.
*   `continue`: Пропускает текущую итерацию и переходит к следующей.

# 10. Отладка и Отладчик в JavaScript

Отладка — процесс поиска и исправления ошибок. Самый мощный инструмент — встроенный отладчик в браузере (Chrome DevTools, Firefox Developer Tools).

### Основные инструменты

1.  **`console.log()`**: Базовый вывод информации в консоль.
    ```javascript
    console.log("Значение переменной:", myVar);
    console.error("Произошла ошибка!", errorObj);
    console.table(arrayOfObjects); // Красивый вывод таблицы
    ```

2.  **Ключевое слово `debugger`**:
    При вставке этого слова в код и запуске страницы в режиме разработки (с открытыми инструментами разработчика), выполнение скрипта приостановится в этой строке, как будто там стоит точка останова (breakpoint).
    ```javascript
    function calculate(a, b) {
        debugger; // Execution stops here
        return a + b;
    }
    ```

3.  **Точки останова (Breakpoints) в DevTools**:
    *   Откройте DevTools (F12).
    *   Перейдите во вкладку **Sources** (Источники).
    *   Найдите ваш JS файл.
    *   Кликните на номер строки слева — появится синяя метка. При перезагрузке страницы код остановится здесь.

### Возможности отладчика при остановке:
*   **Step Over (F10)**: Выполнить текущую строку и перейти к следующей (не заходя внутрь функций).
*   **Step Into (F11)**: Зайти внутрь вызываемой функции.
*   **Step Out (Shift+F11)**: Выполнить функцию до конца и выйти из неё.
*   **Resume/Pause (F8)**: Продолжить выполнение или поставить на паузу.
*   **Scope (Область видимости)**: Панель, где видны текущие значения всех переменных в реальном времени.
*   **Watch (Наблюдение)**: Можно добавить конкретные выражения, чтобы следить за их изменением на каждом шаге.
*   **Call Stack (Стек вызовов)**: Показывает цепочку функций, которые привели к текущей точке выполнения (кто кого вызвал).

### Пример сценария отладки:
Допустим, функция возвращает `NaN`, хотя должна возвращать число.
1.  Ставим `debugger` перед возвратом значения.
2.  Обновляем страницу.
3.  Код останавливается.
4.  Наводим курсор на переменные в коде или смотрим панель **Scope**, чтобы увидеть их реальные значения.
5.  Видим, что одна из переменных равна `"10"` (строка), а другая `undefined`.
6.  Смотрим **Call Stack**, чтобы понять, откуда пришли эти данные.
7.  Исправляем код, убираем `debugger`.
