### Задание 3: «Система Доступа и Логирования»
Создайте симуляцию системы проверки доступа для группы пользователей. У вас есть массив объектов пользователей, где у каждого есть свойства: `name` (строка), `role` (строка: "user", "admin", "guest"), `activeSessions` (число) и `lastLogin` (может быть датой, строкой или `undefined`).

**Требования к логике:**
1.  Создайте константу с максимальным лимитом сессий (например, 3).
2.  Пройдите циклом по всем пользователям.
3.  Для каждого пользователя определите его уровень доступа:
    *   Если роль "admin", доступ разрешен всегда.
    *   Если роль "user", доступ разрешен только если `activeSessions` меньше лимита.
    *   Если роль "guest" или любая другая, доступ запрещен.
4.  Если `lastLogin` равен `undefined` или `null`, замените его на строку "Никогда не входил" (используйте подходящие операторы слияния).
5.  Сформируйте новый массив только из тех пользователей, которым доступ **разрешен**.
6.  Для каждого разрешенного пользователя выведите строку лога в формате: `"Пользователь [Имя] ([Роль]) допущен. Сессий: [Число]. Последний вход: [Дата]"`.
7.  Добавьте искусственную ошибку в данные одного из пользователей (например, запишите число в поле имени) и убедитесь, что ваша программа не падает, а корректно обрабатывает типы (преобразует имя в строку).

*Цель: сложные условные конструкции, работа с массивами объектов, преобразование типов, отладка логики условий.*
