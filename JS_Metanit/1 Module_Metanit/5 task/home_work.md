### Задание 5: «Финальный Босс: Симулятор Игровой Сессии»
Разработайте мини-симулятор игровой сессии для списка игроков.
Входные данные: массив объектов игроков со свойствами: `id`, `name`, `score` (может быть строкой "100", числом 100 или `null`), `isVip` (boolean), `multiplier` (число, может быть undefined).

**Алгоритм работы:**
1.  **Инициализация:** Объявите константы для базовых множителей и пороговых значений победы.
2.  **Обработка данных (Цикл + Типы + Условия):**
    *   Пройдитесь по всем игрокам.
    *   Нормализуйте `score`: если `null` или не число -> 0. Если строка -> преобразовать в число.
    *   Нормализуйте `multiplier`: если `undefined` или `null`, использовать оператор `??` для установки значения 1.0. Если множитель меньше 1, принудительно установить 1.0 (защита от багов).
    *   Рассчитайте финальный счет: `score * multiplier`. Если игрок VIP (`isVip === true`), добавьте дополнительные 100 очков к финальному счету.
3.  **Ранжирование (Логика + Массивы):**
    *   Создайте новый массив, содержащий только тех игроков, чей финальный счет строго больше 0.
    *   Отсортируйте этот массив по убыванию счета (можно реализовать простую сортировку пузырьком внутри цикла или использовать встроенную, если знаете, но попробуйте реализовать логику сравнения вручную для практики условий).
4.  **Определение победителя (Тернарный + Условия):**
    *   Определите победителя (первый элемент отсортированного массива).
    *   Если победитель набрал больше 1000 очков, его ранг — "Legend", иначе если больше 500 — "Pro", иначе — "Novice". Используйте цепочку тернарных операторов или `if-else` для присвоения ранга.
5.  **Вывод и Отладка:**
    *   Выведите топ-3 игроков в красивом формате.
    *   В коде предусмотрите ситуацию, когда массив игроков пуст. Программа не должна выдавать ошибок при обращении к свойствам несуществующих элементов (используйте безопасную навигацию или проверки).
    *   **Критическое требование:** Внедрите в расчеты намеренную ошибку деления на ноль или некорректное приведение типа, которое приведет к `NaN`. Запустите скрипт в отладчике, отследите момент появления `NaN` с помощью_watch_ выражений, найдите источник и исправьте код так, чтобы вместо `NaN` выводилось 0.

*Цель: комплексное применение всех тем. Работа с нечистыми данными, сложная бизнес-логика, защита от крашей, активное использование отладчика для поиска скрытых дефектов.*
