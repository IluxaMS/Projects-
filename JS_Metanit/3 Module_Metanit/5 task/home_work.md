### Задание 5: «Финальный Босс: Конструктор Динамических Дашбордов»

**Контекст:**
Вам нужно создать движок для генерации отчетов, который принимает сырые данные, обрабатывает их цепочкой методов, использует spread-операторы для манипуляций и выдает итоговый конфиг для дашборда. Задача объединяет все темы: создание, spread, мутацию/немутацию, наследование и сложные методы.

**Входные данные:**
1.  Массив «сырых» данных (смешанные типы, мусор): `[10, "20", 30, null, "error", 40, 50]`.
2.  Массив коэффициентов для умножения: `[1, 1.5, 2]`.

**Требования к логике (пошагово):**

**Этап 1: Очистка и нормализация (Использование методов)**
*   Напишите функцию `cleanData(rawArray)`, которая:
    *   Фильтрует массив, оставляя только элементы, которые можно преобразовать в конечное число (исключить `null`, строки-"мусор", но строки-"числа" преобразовать).
    *   Подсказка: Используйте `filter` + `map`. Внутри `filter` проверяйте `!isNaN(Number(item)) && item !== null`.
    *   Внутри `map` преобразуйте всё в числа `Number(item)`.
    *   Верните очищенный массив чисел.

**Этап 2: Генерация сценариев (Использование Spread и вложенности)**
*   Создайте функцию `generateScenarios(cleanNumbers, coefficients)`, которая:
    *   Для каждого коэффициента из массива `coefficients` создает новый массив данных, где каждое число из `cleanNumbers` умножено на этот коэффициент.
    *   Используйте `map` по массиву коэффициентов. Внутри — `map` по числам.
    *   Результат должен быть массивом массивов (матрицей). Например: `[[10, 30...], [15, 45...], [20, 60...]]`.
    *   Дополнительно: Создайте один общий массив `allValues`, который объединяет все эти сценарии в один плоский список, используя **spread-оператор** внутри `map` или метод `flat`.

**Этап 3: Расширенный анализ (Использование Фабрики Умного Массива)**
*   Используйте функцию-фабрику `createNumberArray()`, которую вы реализовали в **Задании 4** (или создайте её заново в этом файле по тому же принципу: замыкание + валидация + методы суммы/среднего).
*   Создайте экземпляр умного массива: `const analyticsArray = createNumberArray();`.
*   Добавьте все значения из массива `allValues` (полученного на Этапе 2) в этот умный массив.
    *   **Важно:** Используйте **spread-оператор** при вызове метода push: `analyticsArray.push(...allValues)`. Это проверит, что ваш метод push корректно обрабатывает rest-параметры и валидирует каждый элемент.
*   Вызовите методы `getSum()` и `getAverage()` у объекта `analyticsArray`, чтобы получить глобальную сумму и среднее значение по всем сценариям. Сохраните эти результаты в переменные `globalSum` и `globalAvg`.

**Этап 4: Формирование итогового отчета (Компоновка объекта)**
*   Сформируйте итоговый объект отчета со следующей структурой (используя литерал объекта):
    ```javascript
    const report = {
        meta: {
            totalItems: <кол-во элементов в allValues>, // Используйте свойство length массива allValues
            generatedAt: <текущая дата (new Date().toISOString())>
        },
        stats: {
            globalSum: globalSum, // Значение, полученное из метода фабрики на Этапе 3
            globalAvg: globalAvg, // Значение, полученное из метода фабрики на Этапе 3
            scenariosCount: <кол-во сценариев> // Длина массива сценариев из Этапа 2
        },
        topValues: <массив из 3-х самых больших чисел из allValues>
    };
    ```
*   Для получения свойства `topValues`:
    *   Создайте копию массива `allValues` (используя spread-оператор `[...allValues]`), чтобы не мутировать оригинал.
    *   Отсортируйте копию по убыванию (используйте `sort` с функцией сравнения `(a, b) => b - a`).
    *   Возьмите первые 3 элемента отсортированного массива (используйте `slice(0, 3)`). Если элементов меньше 3, возьмите все доступные.

**Этап 5: Вывод**
*   Выведите итоговый объект в консоль. Желательно использовать `console.table()` для наглядности, если браузер позволяет, или просто `console.log` с форматированием.

**Ожидаемый результат:**
Полностью автоматизированный пайплайн: от грязных данных до структурированного отчета с использованием кастомного класса, spread-операторов и цепочки методов массивов. Код должен быть модульным (разбит на функции).

*Цель: Комплексное применение всех тем. Очистка данных (`filter`/`map`), работа с матрицами, `flat`/`spread`, наследование (`class`), агрегация (`reduce` внутри класса), финальная компоновка объекта.*
