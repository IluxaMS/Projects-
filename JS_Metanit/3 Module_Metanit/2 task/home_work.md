### Задание 2: «Анализатор Финансовых Транзакций»

**Контекст:**
Вы работаете с выпиской по банковскому счету. Данные представлены как массив объектов, где каждый объект — это транзакция. Вам нужно провести анализ расходов и доходов, используя функциональные методы массивов.

**Входные данные:**
Массив объектов транзакций. Каждый объект имеет свойства:
*   `id` (число): уникальный номер.
*   `type` (строка): `"income"` (доход) или `"expense"` (расход).
*   `amount` (число): сумма транзакции.
*   `category` (строка): категория (например, "Еда", "Зарплата", "Транспорт").

Пример данных:
```javascript
const transactions = [
    { id: 1, type: "income", amount: 5000, category: "Зарплата" },
    { id: 2, type: "expense", amount: 150, category: "Еда" },
    { id: 3, type: "expense", amount: 300, category: "Транспорт" },
    { id: 4, type: "income", amount: 1000, category: "Фриланс" },
    { id: 5, type: "expense", amount: 50, category: "Еда" }
];
```

**Требования к логике (пошагово):**
1.  **Фильтрация расходов:**
    *   Создайте новый массив, содержащий **только** транзакции типа `"expense"`. Используйте метод `filter`.
2.  **Расчет общей суммы расходов:**
    *   Посчитайте общую сумму всех расходов из отфильтрованного массива.
    *   Используйте метод `reduce`. Начальное значение аккумулятора должно быть `0`.
3.  **Поиск самой крупной покупки:**
    *   Найдите объект транзакции среди расходов, у которого максимальная сумма (`amount`).
    *   Используйте метод `reduce` или комбинацию `sort` и взятия первого элемента (но `reduce` эффективнее для этой задачи).
    *   Если расходов нет, результат должен быть `null` или сообщение об этом.
4.  **Преобразование данных (Map):**
    *   Создайте новый массив строк на основе всех транзакций (и доходов, и расходов).
    *   Формат строки: `"[Тип] Категория: Сумма"`. Например: `"expense Еда: 150"`.
    *   Используйте метод `map`.
5.  **Проверка наличия категории:**
    *   Проверьте, есть ли в исходном массиве хотя бы одна транзакция с категорией `"Развлечения"`.
    *   Используйте метод `some`. Выведите `true` или `false`.
6.  **Проверка всех транзакций на положительность:**
    *   Убедитесь, что во всем массиве `amount` больше 0 для каждой транзакции.
    *   Используйте метод `every`.

**Ожидаемый результат:**
Выведите в консоль:
*   Массив только расходов.
*   Общую сумму расходов.
*   Объект самой крупной покупки (или сообщение, если их нет).
*   Массив отформатированных строк.
*   Результаты проверок `some` и `every`.

*Цель: Глубокая отработка методов `filter`, `reduce`, `map`, `some`, `every` на реальных данных.*
