### Задание 4 (Исправленное): «Фабрика Умных Массивов с Валидацией»

**Контекст:**
Поскольку мы пока не используем классы, мы создадим «Умный Массив» с помощью функции-фабрики и замыкания. Эта функция будет возвращать объект, который ведет себя как массив (имеет методы `push`, `pop`, `length` и т.д.), но добавляет дополнительную логику: он разрешает добавлять только числа и умеет сам считать свою сумму.

**Входные данные:**
Отсутствуют (функция создает внутреннее состояние сама).

**Требования к логике (пошагово):**

1.  **Создание функции-фабрики:**
    *   Напишите функцию `createNumberArray()`.
    *   Внутри неё объявите приватную переменную (через `let` или `const`), которая будет хранить настоящий массив данных. Назовите её, например, `storage`. Инициализируйте её пустым массивом `[]`.
    *   Эта переменная должна быть недоступна извне напрямую (инкапсуляция через замыкание).

2.  **Реализация метода `push` с валидацией:**
    *   Внутри функции-фабрики создайте метод `push` (это может быть свойство возвращаемого объекта, содержащее функцию).
    *   Этот метод должен принимать любое количество аргументов (используйте rest-параметры `...items`).
    *   **Логика валидации:**
        *   Пройдитесь циклом или используйте метод массива (например, `forEach`) по переданным аргументам.
        *   Для каждого элемента проверьте: является ли он числом (`typeof item === 'number'`) и не является ли он `NaN`.
        *   Если элемент валиден: добавьте его во внутренний массив `storage` используя стандартный метод `.push()`.
        *   Если элемент НЕ валиден (строка, null, объект и т.д.): **не добавляйте** его в массив и выведите предупреждение в консоль: `"Ошибка отклонения: значение [значение] не является числом"`.
    *   Метод должен возвращать текущую длину внутреннего массива `storage.length`.

3.  **Реализация метода получения данных:**
    *   Создайте метод `getData()`, который возвращает **копию** внутреннего массива `storage`.
    *   **Важно:** Используйте **spread-оператор** (`[...storage]`) для создания копии, чтобы внешний код не мог случайно изменить внутренний массив напрямую через ссылку.

4.  **Реализация кастомных методов расчета:**
    *   Создайте метод `getSum()`, который возвращает сумму всех чисел во внутреннем массиве. Используйте метод `reduce`.
    *   Создайте метод `getAverage()`, который возвращает среднее арифметическое. Если массив пуст, верните `0`.

5.  **Делегирование других методов (Опционально, но желательно):**
    *   Чтобы массив был полезным, добавьте метод `map(callback)`, который применяет функцию к элементам внутреннего массива и возвращает новый массив (используя `storage.map(...)`).
    *   Добавьте метод `getLength()`, возвращающий длину.

6.  **Возврат интерфейса:**
    *   Функция `createNumberArray` должна возвращать объект, содержащий все созданные методы: `{ push, getData, getSum, getAverage, map, getLength }`.

7.  **Тестирование:**
    *   Вызовите фабрику: `const myNums = createNumberArray();`.
    *   Попробуйте добавить числа: `myNums.push(10, 20, 30);`.
    *   Попробуйте добавить мусор: `myNums.push("строка", null, 40, false, NaN);`.
    *   Проверьте содержимое через `myNums.getData()`. Там должны быть только числа: `[10, 20, 30, 40]`.
    *   Вызовите `myNums.getSum()` и `myNums.getAverage()`, выведите результаты.
    *   Проверьте работу `myNums.map(n => n * 2)`. Убедитесь, что исходный массив внутри фабрики не изменился (так как `map` должен возвращать новый массив, а не менять `storage`).

**Ожидаемый результат:**
Работающая функция-фабрика, создающая объекты с поведением «умного массива». Данные защищены от некорректных типов, расчеты работают автоматически, внутреннее состояние скрыто от прямого изменения извне.

*Цель: Закрепление тем «Замыкания» (для приватности), «Функции» (фабричный паттерн), «Rest-параметры», «Методы массивов» (filter/map/reduce внутри методов), «Spread-оператор» (для безопасного возврата копии). Без использования синтаксиса `class`.*
