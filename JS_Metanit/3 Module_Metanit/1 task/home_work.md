### Задание 1: «Умный Менеджер Списка Покупок»

**Контекст:**
Вам необходимо создать систему управления списком покупок. Исходные данные могут приходить в разном формате, и ваша задача — привести их к единому виду, выполнить базовые операции и сформировать отчет.

**Входные данные:**
1.  Базовый список товаров (массив строк): `["Хлеб", "Молоко"]`.
2.  Дополнительный список товаров, который нужно добавить (массив строк): `["Яйца", "Сыр", "Хлеб"]` (обратите внимание, "Хлеб" повторяется).
3.  Список товаров для удаления (массив строк): `["Молоко"]`.

**Требования к логике (пошагово):**
1.  **Создание и объединение:**
    *   Создайте новый массив, который будет содержать все товары из базового списка и дополнительного списка.
    *   **Обязательно** используйте **spread-оператор (`...`)** для объединения этих двух массивов в один новый. Не используйте метод `concat` или цикл для этого шага.
2.  **Удаление дубликатов:**
    *   Обработайте объединенный массив так, чтобы в нем остались только уникальные названия товаров. Повторяющиеся значения (как "Хлеб" в примере) должны быть удалены.
    *   Результат сохраните в новую переменную. (Подсказка: вспомните структуру данных `Set`, которую мы упоминали в контексте итерируемых объектов, или реализуйте логику фильтрации вручную через цикл/методы).
3.  **Удаление конкретных элементов:**
    *   Из массива с уникальными товарами удалите те товары, которые перечислены в списке «на удаление».
    *   Используйте метод массива, который создает **новый** массив без указанных элементов (не изменяйте исходный массив с уникальными товарами напрямую, если это возможно, или используйте мутацию осознанно).
4.  **Сортировка:**
    *   Отсортируйте итоговый список товаров в алфавитном порядке.
    *   **Важно:** Убедитесь, что сортировка работает корректно для строк (стандартное поведение `sort()`).
5.  **Формирование отчета:**
    *   Преобразуйте итоговый отсортированный массив в одну строку, где товары разделены запятой и пробелом (например: "Сыр, Хлеб, Яйца"). Используйте соответствующий метод массива.
    *   Выведите в консоль количество товаров в финальном списке и саму строку-отчет.

**Ожидаемый результат:**
В консоли должно быть выведено количество уникальных оставшихся товаров и строка с их названиями через запятую. Исходные массивы не должны быть потеряны (если вы использовали мутацию, убедитесь, что это оправдано).

*Цель: Отработать создание массивов, spread-оператор, удаление дубликатов, методы `filter`/`includes`, `sort`, `join`.*
