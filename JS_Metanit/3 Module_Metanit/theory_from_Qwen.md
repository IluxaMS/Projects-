# 1. Создание массива и объект `Array`

В JavaScript массивы являются частным случаем объектов. Они упорядочены, имеют специальные свойства для работы с длиной и индексами.

### Способы создания

**1. Литерал массива (Рекомендуемый способ)**
Самый быстрый, читаемый и безопасный способ.
```javascript
const fruits = ["Яблоко", "Груша", "Слива"];
const mixed = [1, "два", true, null, { key: "value" }, [1, 2]]; // Массив может содержать любые типы, включая другие массивы
const empty = [];
```

**2. Конструктор `new Array()`**
Использование конструктора имеет **коварные особенности** в зависимости от количества аргументов.

*   **Один числовой аргумент:** Создает массив указанной длины, заполненный пустыми слотами (`empty slots`). Это **не** то же самое, что массив с элементами `undefined`.
    ```javascript
    const arr1 = new Array(5); 
    console.log(arr1); // [ <5 empty items> ]
    console.log(arr1.length); // 5
    console.log(arr1[0]); // undefined, но ключа "0" не существует!
    
    // Методы вроде map() пропустят пустые слоты!
    const mapped = arr1.map(() => 1); 
    console.log(mapped); // [ <5 empty items> ] (не сработало)
    ```

*   **Несколько аргументов (или один нечисловой):** Создает массив с переданными элементами.
    ```javascript
    const arr2 = new Array(1, 2, 3); // [1, 2, 3]
    const arr3 = new Array("5");     // ["5"] (строка, поэтому считается элементом)
    ```

> **Совет Мой Господин:** Всегда используйте литерал `[]`. Конструктор `new Array()` создает путаницу и потенциальные баги с пустыми слотами.

### Массив как объект
Массив — это объект, у которого:
*   Ключами являются индексы (строковые представления чисел: `"0"`, `"1"`...).
*   Есть специальное свойство `length`, которое обновляется автоматически.
*   Прототипом является `Array.prototype`, откуда берутся все методы (`push`, `map`, `filter` и т.д.).

```javascript
const arr = ["A", "B"];
console.log(typeof arr); // "object"
console.log(arr instanceof Array); // true
console.log(arr.length); // 2

// Можно добавлять произвольные свойства (но не рекомендуется)
arr.customProp = "Hello";
console.log(arr.customProp); // "Hello"
// Но length и итерация по циклу for...of их не увидят
```

# 2. Массивы и Spread-оператор (`...`)

Оператор распространения (Spread Operator) позволяет «распаковать» итерируемый объект (массив, строку, Set, Map) в отдельные элементы.

### Основные применения

**1. Копирование массива (Поверхностная копия)**
Присваивание по ссылке копирует ссылку. Spread создает новый массив с теми же элементами.
```javascript
const original = [1, 2, 3];
const copy = [...original];

copy.push(4);
console.log(original); // [1, 2, 3] (не изменился)
console.log(copy);     // [1, 2, 3, 4]
```
*Важно:* Копия поверхностная. Если внутри массива есть объекты, они скопируются по ссылке.

**2. Объединение массивов**
```javascript
const part1 = [1, 2];
const part2 = [3, 4];
const combined = [...part1, ...part2, 5, 6]; 
// [1, 2, 3, 4, 5, 6]
```

**3. Передача элементов как аргументов функции**
```javascript
const numbers = [5, 10, 15];

function sum(a, b, c) {
    return a + b + c;
}

console.log(sum(...numbers)); // 30 (эквивалентно sum(5, 10, 15))
console.log(Math.max(...numbers)); // 15
```

**4. Вставка элементов в середину**
```javascript
const start = [1, 2];
const end = [5, 6];
const middle = [3, 4];

const result = [...start, ...middle, ...end];
// [1, 2, 3, 4, 5, 6]
```

**5. Преобразование строки в массив символов**
```javascript
const str = "Hello";
const chars = [...str]; 
// ['H', 'e', 'l', 'l', 'o']
// Работает корректно даже с эмодзи, в отличие от str.split('')
```

# 3. Операции с массивами (Мутация vs Немутация)

Операции с массивами делятся на два типа: те, которые изменяют исходный массив (**мутирующие**), и те, которые возвращают новый (**немутующие**).

### Мутирующие методы (Изменяют оригинал)
Используйте осторожно, особенно в React или функциональном программировании.
*   `push(item)` / `pop()`: Добавление/удаление в конец.
*   `unshift(item)` / `shift()`: Добавление/удаление в начало.
*   `splice(index, count, ...items)`: Универсальный метод для вставки, удаления и замены элементов.
*   `sort()`: Сортировка (изменяет порядок в оригинале).
*   `reverse()`: Разворот массива.

```javascript
const arr = [1, 2, 3];
arr.push(4); // arr стал [1, 2, 3, 4]
arr.splice(1, 1, 99); // Удаляет 1 элемент с индекса 1, вставляет 99. arr: [1, 99, 3, 4]
```

### Немутующие методы (Создают новый массив)
Безопасны для использования.
*   `slice(start, end)`: Копирует часть массива.
*   `concat(...)`: Объединяет массивы (устаревает в пользу spread).
*   `map()`, `filter()`, `reduce()`: Методы преобразования (рассмотрены ниже).

```javascript
const original = [1, 2, 3];
const sliced = original.slice(0, 2); // [1, 2]
// original остался [1, 2, 3]
```

# 4. Наследование массивов

Поскольку массивы — это объекты, они участвуют в цепочке прототипов.

### Цепочка прототипов
1.  Экземпляр массива (`[]`) наследует методы от `Array.prototype`.
2.  `Array.prototype` сам является объектом и наследует от `Object.prototype`.
3.  `Object.prototype` имеет `null` в качестве прототипа (конец цепи).

```javascript
const arr = [1, 2];

// Методы массива
console.log(arr.push); // Функция из Array.prototype

// Методы объекта
console.log(arr.toString); // Функция из Object.prototype (доступна массиву)
console.log(arr.hasOwnProperty); // Тоже доступна

// Проверка
console.log(arr instanceof Array); // true
console.log(arr instanceof Object); // true (так как Array наследует Object)
```

### Расширение функциональности (Не рекомендуется в продакшене)
Вы можете добавить свои методы в `Array.prototype`, и они станут доступны всем массивам.
```javascript
Array.prototype.first = function() {
    return this[0];
};

const nums = [10, 20, 30];
console.log(nums.first()); // 10
```
> **Предупреждение:** Изменение встроенных прототипов считается плохой практикой, так как может конфликтовать с будущими обновлениями языка или библиотеками. Лучше создавать вспомогательные функции или использовать классы.

### Субклассы массивов (ES6 Class)
Вы можете создать свой класс, наследующий массив, чтобы добавить специфическую логику.
```javascript
class UniqueArray extends Array {
    addUnique(item) {
        if (!this.includes(item)) {
            this.push(item);
        }
        return this;
    }
}

const unique = new UniqueArray();
unique.addUnique(1);
unique.addUnique(1);
unique.addUnique(2);
console.log(unique); // UniqueArray(2) [1, 2]
console.log(unique instanceof Array); // true
```
Важно: Методы встроенного массива (например, `map`, `filter`) в современном JS возвращают экземпляр того же класса (`UniqueArray`), сохраняя ваши кастомные методы.

# 5. Методы массивов

Это самый мощный инструмент разработчика. Методы делятся на несколько категорий.

### А. Итерационные методы (Перебор)

1.  **`forEach(callback)`**
    Выполняет функцию для каждого элемента. Ничего не возвращает (`undefined`). Используется для побочных эффектов (логирование, изменение DOM).
    ```javascript
    [1, 2, 3].forEach((item, index) => {
        console.log(`Индекс ${index}: ${item * 2}`);
    });
    ```

2.  **`map(callback)`**
    Создает **новый массив**, применяя функцию к каждому элементу. Длина нового массива равна длине старого.
    ```javascript
    const nums = [1, 2, 3];
    const squares = nums.map(n => n * n); 
    // [1, 4, 9]
    // nums остался [1, 2, 3]
    ```

3.  **`filter(callback)`**
    Создает **новый массив** только с теми элементами, для которых функция вернула `true`.
    ```javascript
    const nums = [1, 2, 3, 4, 5];
    const evens = nums.filter(n => n % 2 === 0);
    // [2, 4]
    ```

4.  **`some(callback)` и `every(callback)`**
    *   `some`: Возвращает `true`, если **хотя бы один** элемент проходит проверку.
    *   `every`: Возвращает `true`, только если **все** элементы проходят проверку.
    ```javascript
    const nums = [1, 2, 3];
    console.log(nums.some(n => n > 2)); // true (есть 3)
    console.log(nums.every(n => n > 0)); // true (все больше 0)
    console.log(nums.every(n => n > 1)); // false (1 не больше 1)
    ```

5.  **`find(callback)` и `findIndex(callback)`**
    *   `find`: Возвращает **первый найденный элемент** (значение), удовлетворяющий условию, или `undefined`.
    *   `findIndex`: Возвращает **индекс** первого найденного элемента или `-1`.
    ```javascript
    const users = [{id: 1, name: "Alex"}, {id: 2, name: "Bob"}];
    const user = users.find(u => u.id === 2); 
    // {id: 2, name: "Bob"}
    ```

### Б. Методы агрегации

1.  **`reduce(callback, initialValue)`**
    Самый мощный метод. «Сворачивает» массив в одно значение (число, объект, строку).
    *   `accumulator` (аккумулятор): накопленный результат.
    *   `current`: текущий элемент.
    ```javascript
    const nums = [1, 2, 3, 4];
    
    // Сумма
    const sum = nums.reduce((acc, curr) => acc + curr, 0); 
    // 0+1=1, 1+2=3, 3+3=6, 6+4=10 -> 10

    // Группировка (превращение массива в объект)
    const words = ["apple", "banana", "apricot"];
    const grouped = words.reduce((acc, word) => {
        const firstLetter = word[0];
        if (!acc[firstLetter]) acc[firstLetter] = [];
        acc[firstLetter].push(word);
        return acc;
    }, {});
    // { a: ["apple", "apricot"], b: ["banana"] }
    ```

### В. Методы поиска и проверки

*   `includes(value)`: Проверяет наличие значения (возвращает boolean). В отличие от `indexOf`, работает с `NaN`.
*   `indexOf(value)`: Возвращает индекс первого вхождения или `-1`.
*   `lastIndexOf(value)`: Ищет с конца.

### Г. Методы преобразования

*   `join(separator)`: Превращает массив в строку, соединяя элементы разделителем.
    ```javascript
    ["A", "B", "C"].join("-"); // "A-B-C"
    ```
*   `split(separator)`: (Метод строки) Превращает строку в массив.
*   `flat(depth)`: Сглаживает вложенные массивы.
    ```javascript
    [1, [2, [3]]].flat(1); // [1, 2, [3]]
    [1, [2, [3]]].flat(Infinity); // [1, 2, 3]
    ```
*   `flatMap(callback)`: Эквивалент `map()` + `flat(1)`.

### Д. Методы сортировки и изменения

*   `sort(compareFunction)`:
    **Внимание:** Без функции сравнения сортирует элементы как **строки** (по Unicode). "10" будет раньше "2".
    ```javascript
    const nums = [10, 2, 1];
    nums.sort(); // [1, 10, 2] (НЕ ТО, ЧТО МЫ ХОТИМ!)
    
    // Правильная сортировка чисел
    nums.sort((a, b) => a - b); // [1, 2, 10] (по возрастанию)
    nums.sort((a, b) => b - a); // [10, 2, 1] (по убыванию)
    ```
*   `reverse()`: Меняет порядок на обратный.
