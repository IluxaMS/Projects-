### Задание: «Ядро Асинхронного Процессора Задач (Task Processor Core)»

**Контекст:**
Вы разрабатываете ядро системы, которое принимает задачи (функции), управляет их выполнением, кеширует результаты тяжелых вычислений и позволяет гибко настраивать контекст выполнения. Система должна быть модульной и расширяемой.

**Техническое задание:**

1.  **Динамическая Генерация Задач (`new Function`):**
    *   Представьте, что ваша система получает описание логики задачи в виде строки (например, с сервера или из конфигурации).
    *   Реализуйте функцию-фабрику, которая принимает строку с кодом и возвращает исполняемую функцию.
    *   *Важно:* Учтите особенности области видимости таких функций (они не видят локальные переменные внешнего кода). Подумайте, как передать необходимые данные внутрь такой функции, если она изолирована от замыкания.

2.  **Универсальный Кеширующий Декоратор (Декораторы, `call/apply`, Остаточные параметры):**
    *   Напишите универсальную функцию-декоратор `cachingDecorator(func)`.
    *   Этот декоратор должен работать с функциями, имеющими **любое количество аргументов**.
    *   Он должен запоминать результаты вызовов. Если функция вызывается с теми же аргументами повторно, результат должен браться из кеша, а не вычисляться заново.
    *   *Проблема:* Встроенный `Map` использует один ключ. Вам нужно придумать способ превратить набор аргументов (arguments) в уникальный ключ для хеш-таблицы. Используйте заимствование методов массивов или оператор расширения для работы с псевдомассивом `arguments`.
    *   *Контекст:* Декоратор должен корректно работать с методами объектов, сохраняя контекст `this`. Используйте методы явного указания контекста (`call` или `apply`) для проброса `this` и аргументов в исходную функцию.

3.  **Рекурсивный Анализ Структуры Задач (Рекурсия):**
    *   Задачи могут быть организованы в древовидную структуру (группы задач, подгруппы, отдельные задачи).
    *   Пример структуры:
        ```javascript
        let taskTree = {
            groupA: [
                { id: 1, weight: 10 },
                { id: 2, weight: 20 }
            ],
            groupB: {
                subGroupB1: [
                    { id: 3, weight: 5 }
                ]
            }
        };
        ```
    *   Напишите рекурсивную функцию `calculateTotalWeight(tree)`, которая проходит по всей структуре (независимо от глубины вложенности) и суммирует веса (`weight`) всех конечных задач.
    *   Алгоритм должен различать: если это массив/объект с подгруппами — идти глубже (рекурсивный шаг); если это конечная задача — брать вес (база рекурсии).

4.  **Планировщик с Контролем Контекста (`setTimeout`, `bind`, Потеря `this`):**
    *   Создайте объект `Scheduler`, у которого есть метод `run(taskName)`.
    *   Метод `run` должен планировать выполнение задачи через `setTimeout`.
    *   *Проблема:* При прямой передаче метода в `setTimeout` теряется контекст `this`.
    *   Решите эту проблему двумя способами (на выбор или оба для сравнения):
        1.  Используя функцию-обёртку (стрелочную или обычную).
        2.  Используя метод `bind` для жесткой привязки контекста.
    *   Дополнительно: Реализуйте возможность отмены запланированной задачи (используйте `clearTimeout`).

5.  **Счетчик Вызовов и Именованное Функциональное Выражение (NFE, Свойства функции):**
    *   Возьмите одну из ваших основных функций (например, кеширующую обертку или рекурсивный счетчик).
    *   Добавьте ей свойство-счетчик `callCount`, которое увеличивается при каждом вызове.
    *   Обеспечьте надежность рекурсивных вызовов внутри этой функции, используя **Named Function Expression (NFE)**. Функция должна вызывать саму себя через внутреннее имя, чтобы не зависеть от внешней переменной, которая может быть перезаписана.

6.  **Частичное Применение (Bind, Частичное применение):**
    *   У вас есть базовая функция формирования отчета: `generateReport(prefix, data, suffix)`.
    *   Создайте на её основе специализированную функцию `generateErrorReport`, используя `bind`, где `prefix` уже жестко зафиксирован как `"ERROR: "`, а `suffix` как `" [END]"`.
    *   Новая функция должна принимать только `data`.

**Требования к реализации:**
*   Использовать `"use strict"`.
*   Код должен быть разбит на логические блоки/функции.
*   Не использовать классы (так как они не изучены), использовать объекты и функции-конструкторы или фабричные функции там, где нужно состояние.
*   Демонстрировать понимание того, как работают замыкания (почему декоратор "помнит" кеш).
*   Демонстрировать понимание разницы между `arguments` и реальным массивом.

**Цель задания:**
Показать, что вы умеете комбинировать низкоуровневые механизмы JavaScript (контекст выполнения, лексическое окружение, прототипные методы) для создания высокоуровневых абстракций (декораторы, планировщики).
