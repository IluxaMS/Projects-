### Задание 1: «Аналитический Движок Торговой Платформы»

**Контекст:**
Вы разрабатываете бэкенд-модуль (в виде JS-скрипта) для анализа торговых операций. У вас есть «сырые» данные в виде строк и чисел с погрешностями, которые нужно очистить, структурировать и проанализировать. Данные поступают от разных источников, поэтому формат может быть нестабильным.

**Техническое задание:**

1.  **Парсинг и Очистка «Грязных» Данных (Числа и Строки):**
    *   У вас есть массив «сырых» транзакций, где суммы записаны в разных форматах: некоторые с валютой (`"$120.50"`), некоторые с пробелами (`" 300 "`), некоторые с ошибками (`"NaN"` или `"abc"`).
    *   Напишите функцию, которая принимает этот массив строк и преобразует его в массив чистых чисел.
    *   Используйте методы извлечения чисел из строк (учтите, что некоторые значения могут иметь текст после числа).
    *   Если значение не удается преобразовать в конечное число, оно должно быть заменено на `0`. Используйте проверки на корректность числовых значений.
    *   *Подсказка мышления:* Как отличить "плохое" число от хорошего? Как извлечь число из строки вида "100px" или "$50"?

2.  **Борьба с Неточностью Вычислений (Округление):**
    *   После очистки у вас есть массив чисел с плавающей точкой (например, `0.1 + 0.2`).
    *   Реализуйте функцию суммирования всех транзакций.
    *   Результат сложения должен быть точным до двух знаков после запятой. Избежите проблемы `0.30000000004`.
    *   Результат должен оставаться числом, а не строкой.

3.  **Структурирование Данных (Map и Object):**
    *   У вас есть список категорий товаров (строки) и соответствующие им налоговые ставки (числа).
    *   Создайте коллекцию, которая позволяет эффективно хранить пары «Категория -> Ставка», где ключом может быть любая строка, и порядок добавления важен.
    *   Преобразуйте эту коллекцию в обычный объект JavaScript для совместимости с legacy-кодом, а затем обратно в исходную коллекцию.
    *   *Цель:* Продемонстрировать умение конвертировать структуры данных туда и обратно.

4.  **Уникальные Покупатели (Set):**
    *   У вас есть массив имен покупателей, где имена могут дублироваться (один покупатель совершил несколько покупок).
    *   Создайте структуру данных, которая хранит только уникальные имена.
    *   Выведите список уникальных покупателей.

5.  **Анализ Строк (Поиск и Сравнение):**
    *   У вас есть массив описаний транзакций (строки).
    *   Найдите все описания, которые начинаются с слова "Покупка" или заканчиваются на "USD".
    *   Отсортируйте эти описания в алфавитном порядке, используя корректное языковое сравнение (учтите, что могут быть спецсимволы или разные регистры).
    *   Проверьте, содержится ли в каком-либо описании подстрока "VIP". Если да, выведите позицию первого вхождения. Если нет, используйте трюк с побитовым оператором для компактной проверки (или просто логический вывод).

6.  **Деструктуризация Конфигурации:**
    *   У вас есть объект конфигурации с множеством параметров (валюта, язык, лимиты и т.д.).
    *   Используя деструктуризацию, извлеките только `currency` и `language`.
    *   Если какие-то параметры отсутствуют, присвойте им значения по умолчанию.
    *   Соберите остальные неиспользованные параметры в отдельный объект `restConfig`.

**Требования к реализации:**
*   Использовать `use strict`.
*   Все данные должны быть жестко заданы в коде (без `prompt/alert`), чтобы вы могли сосредоточиться на логике.
*   Использовать функции (стрелочные или обычные) для модульности.
