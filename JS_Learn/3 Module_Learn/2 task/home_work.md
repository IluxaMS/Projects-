### Задание 2: «Система Управления Очередью Задач (Task Manager Core)»

**Контекст:**
Вы пишете ядро системы управления задачами. Задачи имеют приоритет, статус и исполнителя. Система должна позволять добавлять задачи, удалять их, менять порядок, фильтровать и трансформировать списки задач.

**Техническое задание:**

1.  **База Задач и Манипуляции с Массивом (Push/Shift/Splice/Slice):**
    *   Создайте массив объектов-задач. Каждая задача — это объект `{ id, title, priority, status }`.
    *   Реализуйте функцию добавления новой задачи в конец списка.
    *   Реализуйте функцию удаления первой выполненной задачи из начала списка.
    *   Реализуйте функцию, которая позволяет удалить задачу по индексу и сразу вставить на её место две новые срочные задачи (используйте универсальный метод изменения массива).
    *   Создайте копию текущего списка задач (не ссылку!), чтобы можно было работать с ней, не меняя оригинал.

2.  **Поиск и Фильтрация (Find/Filter/IndexOf):**
    *   Найдите первую задачу с приоритетом "High".
    *   Найдите индекс последней задачи со статусом "Pending".
    *   Отфильтруйте все задачи, у которых приоритет не равен "Low".
    *   Проверьте, есть ли в списке задача с определенным ID, используя метод, который возвращает булево значение.

3.  **Трансформация Данных (Map/Reduce/Sort):**
    *   Преобразуйте массив объектов-задач в массив строк формата `"ID: Title [Priority]"`.
    *   Рассчитайте общую «стоимость» выполнения всех задач, где стоимость зависит от приоритета (High=10, Medium=5, Low=1). Используйте метод аккумуляции.
    *   Отсортируйте задачи по приоритету (High -> Medium -> Low). *Внимание:* При сортировке строк помните о лексикографическом порядке, возможно, потребуется функция сравнения.
    *   Переверните итоговый отсортированный список.

4.  **Работа со Строками Заголовков (Slice/Substr/Case):**
    *   У всех заголовков задач приведите первый символ к верхнему регистру, а остальные к нижнему.
    *   Если заголовок длиннее 20 символов, обрежьте его до 20 символов и добавьте "...". Используйте методы извлечения подстрок.
    *   Получите последний символ каждого заголовка, используя современный синтаксис доступа по отрицательному индексу.

5.  **Группировка Исполнителей (Map/Set/Destructuring):**
    *   У вас есть массив имен исполнителей (могут быть дубликаты).
    *   Используя `Set`, получите список уникальных исполнителей.
    *   Создайте `Map`, где ключом будет имя исполнителя, а значением — количество его задач (посчитайте вручную через цикл или методы массива).
    *   Переберите этот `Map` и выведите статистику.
    *   Используйте деструктуризацию массива внутри цикла перебора, чтобы красиво вывести имя и количество.

6.  **Конструктор Задач и Опциональная Цепочка:**
    *   Напишите функцию-конструктор `Task(title, assignee)`.
    *   Внутри конструктора создайте объект `meta`, который может быть `null` (если метаданных нет).
    *   Добавьте метод `getAssigneeName()`, который безопасно возвращает имя исполнителя, даже если поле `assignee` отсутствует или равно `null` (используйте `?.`).
    *   Создайте несколько экземпляров задач через `new`.

7.  **Спецсимволы и Длина:**
    *   В одном из заголовков задачи используйте спецсимвол перевода строки `\n` и табуляции `\t`.
    *   Выведите реальную длину этой строки (учтите, что `\n` — это один символ).
    *   Проверьте, начинается ли заголовок с табуляции.

**Требования к реализации:**
*   Акцент на методах массивов: `map`, `filter`, `reduce`, `sort`, `splice`, `slice`.
*   Акцент на методах строк: `slice`, `toUpperCase`, `startsWith`.
*   Использование `Map` и `Set` по назначению.
*   Деструктуризация в параметрах функций или при присваивании.
