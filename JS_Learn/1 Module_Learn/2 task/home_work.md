### Задание 2: «Генератор Паролей и Менеджер Безопасности»

**Описание:**
Разработайте утилиту для генерации надежных паролей и проверки их сложности. Программа должна работать в цикле, позволяя пользователю генерировать пароли до тех пор, пока он не решит выйти, и оценивать качество сгенерированных или введенных паролей.

**Требования к функционалу:**

1.  **Структура и Константы:**
    *   Включите строгий режим.
    *   Определите набор символов для генерации пароля (строчные буквы, заглавные буквы, цифры, спецсимволы) в виде константных строк.
    *   Используйте камелКейс для именования переменных и функций.

2.  **Основной цикл взаимодействия:**
    *   Организуйте бесконечный цикл, который предлагает пользователю выбрать действие: «Сгенерировать пароль» или «Проверить свой пароль» или «Выход».
    *   Для выхода из цикла используйте директиву прерывания, основанную на выборе пользователя.
    *   Если пользователь вводит некорректный вариант, цикл должен начинаться заново, пропуская остальную часть итерации (используйте команду перехода к следующей итерации).

3.  **Генерация пароля (Математика и Случайность):**
    *   Запросите желаемую длину пароля.
    *   Напишите функцию генерации, которая принимает длину и набор символов.
    *   Внутри функции используйте цикл для посимвольного построения строки пароля.
    *   Для выбора случайного символа используйте математические операции (округление, получение остатка от деления для индексации массива/строки).
    *   Реализуйте логику: если длина пароля меньше 8 символов, автоматически увеличьте её до 8, выдав предупреждение.

4.  **Проверка сложности (Логика и Сравнения):**
    *   Напишите отдельную функцию оценки сложности пароля.
    *   Функция должна принимать строку пароля и возвращать оценку (например, от 1 до 5) или текстовый вердикт («Слабый», «Средний», «Надежный»).
    *   Критерии оценки:
        *   Наличие цифр.
        *   Наличие заглавных букв.
        *   Наличие спецсимволов.
        *   Длина более 12 символов.
    *   Используйте логические операторы И (`&&`) и ИЛИ (`||`) для комбинирования условий. Обратите внимание на сокращенное вычисление: если первое условие ложно, дальнейшие проверки могут быть избыточны (оптимизируйте логику).
    *   Используйте оператор нулевого слияния (`??`) для установки оценки по умолчанию, если функция оценки по какой-то причине вернет `null` или `undefined`.

5.  **Работа со строками и Преобразование типов:**
    *   При анализе пароля приводите символы к нижнему или верхнему регистру для упрощения проверок (если будете делать проверки через сравнение).
    *   Используйте методы поиска подстроки или включение символа в строку.
    *   Демонстрируйте понимание разницы между `==` и `===` при сравнении результатов ввода пользователя с ожидаемыми командами меню (учитывая, что `prompt` всегда возвращает строку).

6.  **Функции-колбэки и Модульность:**
    *   Реализуйте функцию `processUserChoice`, которая принимает выбор пользователя и две функции-колбэка: одну для генерации, другую для проверки.
    *   Внутри `processUserChoice` вызывайте соответствующий колбэк в зависимости от выбора.
    *   Это продемонстрирует умение передавать функции как аргументы.

7.  **Финальный аккорд:**
    *   После выхода из цикла попрощайтесь с пользователем, используя его имя (если оно было введено ранее, иначе используйте «Пользователь»).
    *   Выведите в консоль статистику сеанса: сколько паролей было сгенерировано и сколько проверено. Для этого используйте внешние переменные-счетчики, которые инкрементируются внутри функций.

**Критерии успеха:**
*   Цикл работает корректно, выход осуществляется только по команде.
*   Функции переиспользуемы и имеют понятные имена.
*   Логика оценки пароля покрывает все комбинации условий.
*   Используются как Function Declaration, так и Function Expression (или стрелочные функции).
*   Код чистый, с комментариями, объясняющими сложную логику.
