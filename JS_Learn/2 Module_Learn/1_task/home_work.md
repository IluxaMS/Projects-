### Задание 1: «Система Учета Сотрудников с Динамическими Ролями»

**Контекст:**
Вы разрабатываете модуль для HR-системы. Вам нужно хранить данные о сотрудниках, но структура данных может меняться. У разных сотрудников разные наборы дополнительных свойств (кто-то имеет машину, кто-то нет; у кого-то есть кабинет, у кого-то удаленка). Ваша задача — создать гибкую систему хранения и обработки этих данных, используя объекты.

**Техническое задание:**

1.  **Создание базы данных (Объекты):**
    *   Создайте объект-контейнер (назовите его, например, `companyStaff`), который будет хранить данные о сотрудниках.
    *   Ключами этого объекта должны быть уникальные ID сотрудников (строки), а значениями — объекты с данными сотрудника.
    *   Реализуйте функцию-конструктор `Employee(name, position, salary)`.
        *   Она должна создавать объект со свойствами `name`, `position`, `salary`.
        *   Добавьте метод `getInfo()`, который возвращает строку с именем и должностью, используя `this`.
    *   Создайте минимум 3 разных сотрудника, используя оператор `new`, и поместите их в объект `companyStaff` под разными ключами (ID).

2.  **Динамическое расширение (Добавление свойств):**
    *   Для одного из сотрудников добавьте свойство `car` (объект `{ brand: "BMW", year: 2020 }`).
    *   Для другого добавьте свойство `office` (строка "Room 101").
    *   Третий сотрудник не должен иметь никаких дополнительных свойств, кроме базовых.
    *   *Важно:* Не меняйте конструктор для всех, добавляйте свойства индивидуально конкретным экземплярам, чтобы продемонстрировать гибкость объектов.

3.  **Безопасный доступ (Опциональная цепочка):**
    *   Напишите функцию `checkBenefit(employeeId)`, которая принимает ID сотрудника.
    *   Функция должна проверить, есть ли у сотрудника машина.
    *   Если машина есть, вывести марку. Если нет — вывести "Нет служебного авто".
    *   **Критическое требование:** Используйте опциональную цепочку `?.` для доступа к свойству `brand`. Код не должен падать с ошибкой, если у сотрудника вообще нет свойства `car` или если самого сотрудника с таким ID нет в базе (возвращает `undefined`).

4.  **Копирование и Изоляция (Клонирование):**
    *   Руководство решило создать "теневого" сотрудника для тестов — полную копию первого сотрудника, но с измененным именем.
    *   Реализуйте поверхностное клонирование объекта первого сотрудника в новый объект `testEmployee`.
    *   Используйте цикл `for..in` для копирования свойств (не используйте `Object.assign`, так как цель — понять механизм копирования через цикл, описанный в теории).
    *   Измените имя у `testEmployee`. Убедитесь (выведя в alert/console), что имя оригинального сотрудника **не** изменилось.
    *   *Проверка понимания:* Если бы у сотрудника было вложенное свойство-объект (например, та же машина), изменилось бы оно в оригинале при изменении в копии? Напишите комментарий в коде, объясняющий, почему так происходит, исходя из теории ссылок.

5.  **Перебор и Отчетность:**
    *   Напишите функцию `printAllStaff()`, которая перебирает объект `companyStaff` с помощью цикла `for..in`.
    *   Для каждого сотрудника вызовите его метод `getInfo()` и выведите результат.

**Цель задания:** Научиться работать с объектами как с коллекциями, использовать `this` в методах, безопасно обращаться к вложенным свойствам через `?.` и понимать разницу между ссылкой и копией.
