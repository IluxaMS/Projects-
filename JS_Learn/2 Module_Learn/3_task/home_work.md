### Задание 3: «Умный Дом: Управление Устройствами через Единый Хаб»

**Контекст:**
Вы создаете систему управления умным домом. Все устройства (лампы, термостаты, камеры) хранятся в едином объекте-хабе. Устройства могут быть разных типов и иметь разные настройки. Некоторые устройства могут быть временно отключены (null/undefined). Система должна быть устойчива к отсутствию устройств.

**Техническое задание:**

1.  **Структура Хаба (Объект объектов):**
    *   Создайте объект `smartHomeHub`.
    *   Добавьте в него свойства, представляющие комнаты: `kitchen`, `bedroom`, `hall`.
    *   Каждая комната — это объект, который может содержать устройства.
    *   Например: `smartHomeHub.kitchen = { lamp: { isOn: false, brightness: 50 }, thermostat: { temp: 22 } }`.
    *   Оставьте `smartHomeHub.bedroom` пустым объектом `{}`.
    *   `smartHomeHub.hall` сделайте равным `null` (представим, что датчик в коридоре сломан и данные недоступны).

2.  **Универсальный переключатель (Опциональная цепочка ?.):**
    *   Напишите функцию `toggleDevice(roomName, deviceName)`.
    *   Функция должна пытаться переключить состояние `isOn` устройства на противоположное.
    *   **Главное требование:** Функция не должна вызывать ошибок, если:
        *   Комнаты не существует.
        *   Комната равна `null` (как hall).
        *   Устройства в комнате нет.
    *   Используйте `?.` для доступа к свойствам. Если устройство не найдено, выведите сообщение "Устройство не доступно".

3.  **Методы Устройств (This и Функции в объектах):**
    *   Для лампы на кухне создайте метод `toggle()`, который меняет `this.isOn`.
    *   Для термостата создайте метод `setTemp(newTemp)`, который меняет `this.temp`.
    *   Вызовите эти методы через объект хаба.

4.  **Копирование Конфигурации (Клонирование):**
    *   Вы хотите сделать "Гостевой режим", который копирует текущие настройки кухни, но позволяет их менять, не ломая основные настройки.
    *   Скопируйте объект `smartHomeHub.kitchen` в новый объект `guestModeKitchen`.
    *   Используйте цикл `for..in` для копирования.
    *   Измените яркость лампы в `guestModeKitchen`.
    *   Проверьте, изменилась ли яркость в основном хабе. (Внимание: если внутри лампы есть вложенные объекты, помните про ссылки. В данном случае `brightness` — число, примитив, так что все должно быть ок. Но если бы вы копировали сам объект лампы, будьте внимательны).

5.  **Динамическое добавление (Квадратные скобки):**
    *   Позвольте пользователю через `prompt` ввести название новой комнаты и название устройства в ней.
    *   Динамически добавьте это в `smartHomeHub`, используя квадратные скобки `smartHomeHub[roomName] = ...`.
    *   Если такая комната уже есть, добавьте устройство в существующий объект комнаты.

6.  **Отладка Типов (typeof):**
    *   Напишите функцию `diagnoseHub()`, которая проходит циклом `for..in` по всем комнатам хаба.
    *   Для каждой комнаты выведите её имя и тип значения (`typeof`).
    *   Это поможет увидеть, что `hall` имеет тип "object" (хотя это null, помните про особенность `typeof null` из теории? Нет, в теории сказано `typeof null // "object"`. Используйте это знание). *Поправка:* В теории сказано, что `typeof null` возвращает `"object"`. Выведите это в консоль/alert для комнаты hall.

**Цель задания:** Работа с глубоко вложенными объектами, массовое применение `?.` для защиты от ошибок, динамическое создание свойств объектов, понимание `typeof` для отладки.
