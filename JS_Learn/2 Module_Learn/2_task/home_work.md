### Задание 2: «Конфигуратор Игрового Персонажа (RPG Stats)»

**Контекст:**
Вы пишете движок для текстовой RPG-игры. У персонажей есть характеристики (сила, ловкость, интеллект), которые могут меняться. Также у персонажей есть инвентарь, который представлен объектом. Вам нужно реализовать систему прокачки и проверки условий для использования предметов.

**Техническое задание:**

1.  **Архитектура Персонажа (Конструктор и Методы):**
    *   Создайте конструктор `Hero(name)`.
    *   Внутри конструктора инициализируйте объект `stats` со свойствами: `strength: 10`, `agility: 10`, `intelligence: 10`.
    *   Инициализируйте пустой объект `inventory`.
    *   Добавьте метод `levelUp(statName)`, который увеличивает выбранную характеристику на 5.
        *   `statName` передается как строка (например, "strength").
        *   Используйте квадратные скобки `this.stats[statName]` для доступа к свойству, имя которого хранится в переменной.
        *   Добавьте проверку: если такой характеристики не существует, выведите предупреждение.

2.  **Инвентарь и Ссылки:**
    *   Создайте объект-шаблон предмета `swordTemplate = { type: "weapon", damage: 20, durability: 100 }`.
    *   При добавлении предмета в инвентарь героя (метод `addItem(itemName, itemObject)`), предмет должен сохраняться в `this.inventory`.
    *   **Ловушка:** Добавьте один и тот же объект `swordTemplate` двум разным героям.
    *   Реализуйте метод `attack()` у героя, который уменьшает `durability` текущего оружия на 10.
    *   Проверьте: если первый герой атакует, изменится ли прочность оружия у второго героя? Объясните в комментарии, почему это произошло (ссылки на один объект).

3.  **Безопасная проверка экипировки (?.):**
    *   Реализуйте метод `getWeaponDamage()`.
    *   Он должен возвращать урон текущего оружия.
    *   Если оружия нет, или у оружия нет свойства `damage`, метод должен возвращать 0.
    *   Используйте опциональную цепочку `?.` для безопасного чтения `this.inventory.currentWeapon?.damage`.
    *   *Подсказка:* Вам нужно как-то пометить, какое оружие сейчас в руках. Можно добавить свойство `currentWeapon` в объект `inventory`, которое будет ссылаться на один из предметов в инвентаре, или просто хранить объект оружия напрямую.

4.  **Клонирование Инвентаря (Глубокая vs Поверхностная копия):**
    *   Герой нашел "Зеркало Duplicate". Оно создает копию его инвентаря.
    *   Напишите функцию `cloneInventory(sourceInventory)`, которая создает новый объект и копирует туда все предметы из исходного инвентаря через `for..in`.
    *   Присвойте эту копию новому герою.
    *   Измените прочность оружия в копии. Убедитесь, что у оригинального героя прочность не изменилась (если копирование сделано правильно для верхнего уровня).

5.  **Взаимодействие (Старая теория + Новая):**
    *   Через `prompt` спросите у пользователя, какую характеристику прокачать.
    *   Вызовите метод `levelUp` с введенным значением.
    *   Выведите итоговые статы через `alert`, используя шаблонные строки.

**Цель задания:** Закрепить работу с динамическими ключами `[key]`, понять проблему мутации общих объектов-ссылок, применить `?.` для проверки наличия вложенных свойств.
