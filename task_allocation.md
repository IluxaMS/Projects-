### Полный перечень функций для реализации

Всего необходимо реализовать **17 функций**. Их можно разделить на четыре логические группы:

#### 1. Преобразователи (Converters) — 4 функции
*   `int s21_from_int_to_decimal(int src, s21_decimal *dst)`
*   `int s21_from_float_to_decimal(float src, s21_decimal *dst)`
*   `int s21_from_decimal_to_int(s21_decimal src, int *dst)`
*   `int s21_from_decimal_to_float(s21_decimal src, float *dst)`

#### 2. Арифметические операторы (Arithmetic) — 4 функции
*   `int s21_add(s21_decimal value_1, s21_decimal value_2, s21_decimal *result)`
*   `int s21_sub(s21_decimal value_1, s21_decimal value_2, s21_decimal *result)`
*   `int s21_mul(s21_decimal value_1, s21_decimal value_2, s21_decimal *result)`
*   `int s21_div(s21_decimal value_1, s21_decimal value_2, s21_decimal *result)`

#### 3. Операторы сравнения (Comparison) — 6 функций
*   `int s21_is_less(s21_decimal, s21_decimal)`
*   `int s21_is_less_or_equal(s21_decimal, s21_decimal)`
*   `int s21_is_greater(s21_decimal, s21_decimal)`
*   `int s21_is_greater_or_equal(s21_decimal, s21_decimal)`
*   `int s21_is_equal(s21_decimal, s21_decimal)`
*   `int s21_is_not_equal(s21_decimal, s21_decimal)`

#### 4. Другие функции / Округление (Other/Rounding) — 3 функции
*   `int s21_floor(s21_decimal value, s21_decimal *result)`
*   `int s21_round(s21_decimal value, s21_decimal *result)`
*   `int s21_truncate(s21_decimal value, s21_decimal *result)`
*   *(Бонусная/Вспомогательная, но указанная в таблице)*: `int s21_negate(s21_decimal value, s21_decimal *result)`

*(Примечание: `s21_negate` часто используется как вспомогательная функция внутри арифметики, но по ТЗ она вынесена в отдельный блок "Другие функции". Я включу её в распределение.)*

### Распределение задач между 3 участниками

Для эффективной работы и минимизации конфликтов при слиянии кода, предлагаю следующее разделение, основанное на сложности и взаимозависимости функций.

#### 👤 Участник 1: "Архитектор базовых операций и сравнений"
**Фокус:** Работа с битами, нормализация чисел, логика сравнения. Эти функции являются фундаментом для всей остальной логики.

1.  **Сравнения (6 функций):**
    *   `s21_is_less`
    *   `s21_is_less_or_equal`
    *   `s21_is_greater`
    *   `s21_is_greater_or_equal`
    *   `s21_is_equal`
    *   `s21_is_not_equal`
    *   *Комментарий:* Сравнение требует понимания знака и масштаба (scale). Реализовав их первыми, вы обеспечите тестами базу для отладки арифметики.
2.  **Вспомогательная функция:**
    *   `s21_negate` (инверсия знака). Простая, но критически важная для вычитания.

**Задачи участника 1:**
*   Реализовать парсинг `bits[3]` (знак и scale).
*   Написать функции сравнения мантисс с учетом масштаба.
*   Подготовить общие вспомогательные функции (например, `normalize_scale`, `get_bit`, `set_bit`), которыми смогут пользоваться остальные.

#### 👤 Участник 2: "Мастер арифметики"
**Фокус:** Самая сложная часть проекта. Требует работы с большими числами (96 бит), переносами разрядов и обработкой переполнений.

1.  **Арифметика (4 функции):**
    *   `s21_add` (Сложение)
    *   `s21_sub` (Вычитание — можно реализовать через сложение с `negate`)
    *   `s21_mul` (Умножение)
    *   `s21_div` (Деление)
    *   *Комментарий:* Деление и умножение — самые сложные операции. Требуется реализация банковского округления при переполнении мантиссы.

**Задачи участника 2:**
*   Реализовать сложение/вычитание "в столбик" для 96-битных чисел.
*   Реализовать умножение длинной арифметики.
*   Реализовать деление с остатком.
*   Внедрить логику обработки ошибок (переполнение/underflow) согласно ТЗ (коды возврата 1, 2, 3).

#### 👤 Участник 3: "Конвертер и специалист по округлению"
**Фокус:** Взаимодействие со стандартными типами C (`int`, `float`) и финальная обработка результатов.

1.  **Преобразователи (4 функции):**
    *   `s21_from_int_to_decimal`
    *   `s21_from_float_to_decimal` (Требует внимательной работы с точностью float и банковским округлением до 7 значащих цифр).
    *   `s21_from_decimal_to_int`
    *   `s21_from_decimal_to_float`
2.  **Округление (3 функции):**
    *   `s21_floor`
    *   `s21_round`
    *   `s21_truncate`

**Задачи участника 3:**
*   Разобраться с представлением `float` (IEEE 754) для корректного конвертирования в decimal.
*   Реализовать логику отбрасывания дробной части или округления для `floor/round/truncate`.
*   Обеспечить корректную обработку крайних случаев (очень маленькие числа < 1e-28, бесконечности).

### Рекомендации по процессу разработки

1.  **Общий заголовочный файл (`s21_decimal.h`):**
    *   Создайте его сразу вместе. Определите структуру `s21_decimal` и прототипы всех функций.
2.  **Вспомогательные функции (Helpers):**
    *   Вам понадобятся внутренние функции, не описанные в публичном API, например:
        *   `void set_scale(s21_decimal *num, int scale)`
        *   `int get_scale(s21_decimal num)`
        *   `void shift_left/right(s21_decimal *num)` (для умножения/деления на 10)
        *   `int compare_mantissas(s21_decimal a, s21_decimal b)` (сравнение без учета знака и масштаба, после приведения к общему масштабу)
    *   Договоритесь, кто пишет эти хелперы. Лучше всего это сделать **Участнику 1**, так как он работает с битовой структурой.
3.  **Тестирование (Check):**
    *   Каждый участник пишет тесты для своих функций.
    *   Важно: Арифметику (Участник 2) сложно тестировать изолированно без конвертеров (Участник 3). Поэтому начните с простых кейсов (целые числа) и постепенно усложняйте.
4.  **Стиль кода:**
    *   Не забывайте про Google Style и C11. Используйте `clang-format` для автоматического форматирования.

### План действий на первые дни:

1.  **День 1:**

3.  **День 3:**
    *   Участник 2: Реализует `s21_mul` и `s21_div`.
