## 🔹 Задание 1: Калькулятор через `prompt` и `alert`

### Что требовалось:
> Программа должна принимать **два числа** и выполнять **четыре операции**: сложение, вычитание, умножение, деление.

### Что сделано:
- Принимает **любое количество чисел**, разделённых пробелом.
- Выполняет последовательные операции:  
  `a + b + c...`, `a - b - c...`, `a * b * c...`, `a / b / c...`

👉 Это **не совсем то**, что обычно подразумевается под "калькулятором" в учебных целях. Обычно просят:  
> Введите первое число → Введите второе число → Выберите операцию → Вывести результат.

Но ваш подход **интересный и технически рабочий** — просто нестандартный.

### Ошибки и недочёты:
1. **Деление на ноль не обработано**  
   Если ввести `5 0`, то `division = 5 / 0` → `Infinity`. Лучше проверять.
2. **Умножение при первом числе = 0**  
   Если первое число — `0`, то `multiplication = 0`, и всё произведение будет `0`, даже если остальные числа ненулевые.  
   Но это логично для умножения — так и должно быть. Однако если пользователь введёт `0 5 10`, он может ожидать `50`, а получит `0`.  
   → Это **логическая проблема дизайна**, а не кода.
3. **Лишняя точка с запятой**: `alert("n/a");;` — вторая `;` не нужна (но не ошибка).
4. **Преобразование в строку избыточно**:  
   `${String(addition)}` → можно просто `${addition}`. JS сам преобразует.

### Вердикт:
✅ Работает.  
⚠️ Не соответствует классическому ожиданию от "калькулятора".  
🔧 Требует уточнения условий, но за креативность — плюс.

## 🔹 Задание 2: Игра «Угадай число» (от 1 до 100)

### Что требовалось:
- Компьютер загадывает **случайное число от 1 до 100**.
- Пользователь вводит свои варианты.
- Программа говорит: "больше", "меньше" или "угадал".
- Считать **количество попыток**.
- Цель — угадать **за минимальное число попыток**.

### Что сделано (консольная версия):
```js
const secretNumber = Math.floor(Math.random() * 2) + 1; // ← только 1 или 2!
let x = 0;
let user_Number = 1;

while (x < 10) {
    if (secretNumber === user_Number) {
        console.log("Вы угадали!");
        user_Number = 1; // ← всегда 1?!
    } else {
        console.log("Вы не угадали!");
        user_Number = 2; // ← всегда 2?!
    }
    x++
}
```

🔴 **Критические ошибки**:
1. **Диапазон не от 1 до 100, а от 1 до 2**  
   `Math.random() * 2` → даёт [0, 2), `floor` → 0 или 1, `+1` → 1 или 2.  
   ❌ Нарушено условие задачи.
2. **Пользователь не вводит число!**  
   Вы **жёстко задали** `user_Number = 1`, потом `= 2`.  
   Это **не игра**, а автоматический перебор двух значений.  
   ❌ Нет взаимодействия с пользователем.
3. **Нет подсказок** ("больше"/"меньше").
4. **Нет подсчёта попыток до победы** — просто 10 итераций.

### Что сделано (версия с `prompt`):
```js
const secretNumber = Math.floor(Math.random() * 2) + 1; // ← снова 1–2!
let x = 1;
let user_Number = prompt("Введите число:");

while (x < 11) {
    if (secretNumber === Number(user_Number)) {
        alert(`Вы угадали! Попытка №${x}`);
        user_Number = prompt("Введите число:"); // ← зачем спрашивать после победы?
    } else {
        alert(`Вы не угадали. Попытка №${x}`);
        user_Number = prompt("Введите число:");
    }
    x++
}
```

🔴 **Ошибки**:
1. **Всё ещё диапазон 1–2**, а не 1–100.
2. **Нет подсказок** — пользователь тыкает вслепую.
3. **Игра продолжается даже после победы** — странно.
4. **Нет выхода из цикла при угадывании** — должно быть `break`.

🟢 **Плюсы**:
- Есть `prompt` → пользователь вводит данные.
- Есть счётчик попыток.
- Преобразование `Number(user_Number)` — правильно.

### Как должно быть (пример):
```js
const secret = Math.floor(Math.random() * 100) + 1;
let attempts = 0;
let guess;

while (guess !== secret) {
  guess = Number(prompt("Угадайте число от 1 до 100:"));
  attempts++;

  if (guess < secret) alert("Больше!");
  else if (guess > secret) alert("Меньше!");
  else alert(`Победа! За ${attempts} попыток.`);
}
```

### Вердикт:
❌ Основное требование (диапазон 1–100 + интерактивность + подсказки) **не выполнено**.  
✅ Есть попытка использовать `prompt` — это хорошо.

## 🔹 Задание 3: Чётные числа от 1 до 50

### Ваш код:
```js
for (let i = 1; i < 50; i++) {
    if (i % 2 === 0) {
        console.log(i);
    }
}
```

### Анализ:
- Выводит: 2, 4, 6, ..., **48**
- ❌ **Пропущено число 50**, потому что условие `i < 50`

> Требование: **«от 1 до 50»** → значит, **включая 50**.

### Исправление:
```js
for (let i = 1; i <= 50; i++) { // ← <= вместо <
    if (i % 2 === 0) {
        console.log(i);
    }
}
```

Или ещё лучше — начинать с 2 и шагать по 2:
```js
for (let i = 2; i <= 50; i += 2) {
    console.log(i);
}
```

### Вердикт:
⚠️ Мелкая, но важная ошибка — **50 не выводится**.  
В остальном — идеально.
