## 🔹 Задание 3: Сохранение объекта в `localStorage` и загрузка

### Ваш базовый вариант:
```js
function localstorage_set(array = []) {
    for (let i = 0; i < array.length; i++) {
        localStorage.setItem(`user${i + 1}`, JSON.stringify(array[i]))
    }
}

function localstorage_get(array = []) {
    for (let i = 0; i < array.length; i++) {
        const reverse_data_loading = localStorage.getItem(`user${i + 1}`);
        if (reverse_data_loading) {
            const loaded_data = JSON.parse(reverse_data_loading);
            console.log(loaded_data);
        } else {
            console.log(`Ключ user${i + 1} не найден`);
        }
    }
}
```

### Анализ:
✅ Использованы `JSON.stringify` и `JSON.parse` — верно.  
✅ Обработка случая, когда ключ не найден — хорошо.  
⚠️ **Логическая ошибка**: функция `localstorage_get` принимает **массив объектов**, но использует их **только для подсчёта длины**.  
→ Это **неэффективно**: если вы хотите загрузить **все сохранённые пользователи**, лучше хранить их **в одном ключе как массив**:

```js
// Сохранение:
localStorage.setItem('users', JSON.stringify([user1, user2, user3]));

// Загрузка:
const users = JSON.parse(localStorage.getItem('users') || '[]');
```

Но ваш подход **работает**, просто не оптимален.

🟢 **Вердикт**: 8/10
